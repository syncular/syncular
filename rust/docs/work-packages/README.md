# Work Packages

Each file in this folder is an implementation batch. A batch is done only when
its acceptance criteria, tests, benchmarks, docs, and commit notes are complete.

## Required Sections

Every active work package should include:

- status
- goal
- scope
- acceptance criteria
- required gates
- accept/reject rule when performance-sensitive
- current evidence or baseline
- next action

Later WPs may start thinner, but before a WP becomes `Now` it must be expanded
to this shape.
