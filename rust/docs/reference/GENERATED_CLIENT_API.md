# Generated Client API

This is the app-facing shape the Rust-first client should preserve across
Rust, Browser TypeScript, Swift, Kotlin, and JVM bindings.

## Invariants

- Reads stay query-builder-first. Rust uses Diesel, Browser TypeScript uses
  Kysely, and Swift/Kotlin use the generated lightweight query builder.
- Synced writes go through generated mutations, generated CRDT helpers, blob
  helpers, or conflict helpers. Do not write app tables directly with raw SQL.
- Generated mutation input and patch types omit server-owned columns such as
  `server_version` and CRDT persistence columns such as `title_yjs_state`.
- CRDT state columns remain readable through row types, query builders,
  metadata, and row-delta helpers so apps can observe and debug CRDT state.
- Low-level bindings expose generic JSON methods; app-specific table helpers are
  generated by the app schema.

## Rust

Typed local reads are Diesel queries owned by `client.read(...)`:

```rust
use diesel::prelude::*;
use crate::generated::{diesel_tables::TaskRow, schema};

let tasks: Vec<TaskRow> = client.read(
    schema::tasks::dsl::tasks
        .filter(schema::tasks::dsl::user_id.eq("user-rust"))
        .order(schema::tasks::dsl::server_version.desc())
        .select(TaskRow::as_select()),
)?;
```

Writes use generated mutation DTOs and one outbox path:

```rust
use crate::generated::syncular;
use syncular::prelude::SyncularGeneratedMutationsExt;

let inserted = client.mutations().tasks().insert(
    syncular::NewTask::with_generated_id(
        "Ship Rust",
        "user-rust",
        Some("project-rust"),
    ),
)?;

client
    .mutations()
    .tasks()
    .update(syncular::TaskPatch::new(&inserted.id).completed(1))?;
```

CRDT fields are written through generated CRDT helpers, not by setting the
state column:

```rust
use syncular_client::crdt_yjs::{build_yjs_text_update, BuildYjsTextUpdateArgs};

let update = build_yjs_text_update(BuildYjsTextUpdateArgs {
    previous_state_base64: existing_task.title_yjs_state.clone(),
    next_text: "Edited title".to_string(),
    container_key: Some("title".to_string()),
    update_id: None,
})?;

client.mutations().tasks().update(
    syncular::TaskPatch::new(&existing_task.id)
        .title_yjs_update(update.update),
)?;
```

Rust live queries keep the same query-builder expression:

```rust
let mut live_tasks = client.live_query(["tasks"], || {
    schema::tasks::dsl::tasks
        .filter(schema::tasks::dsl::user_id.eq("user-rust"))
        .order(schema::tasks::dsl::server_version.desc())
        .select(TaskRow::as_select())
})?;

let report = client.sync_http()?;
if live_tasks.refresh_if_changed(&mut client, &report)? {
    render(live_tasks.rows());
}
```

## Browser TypeScript

The generated database is still a Kysely database:

```ts
import { createSyncularAppDatabase } from './generated/syncular.generated';

const database = await createSyncularAppDatabase({
  config: { baseUrl: '/sync', actorId, clientId, projectId },
});

const tasks = await database
  .selectFrom('tasks')
  .select(['id', 'title', 'server_version'])
  .where('user_id', '=', actorId)
  .orderBy('server_version', 'desc')
  .execute();
```

Generated mutation types keep server-owned and CRDT state columns out of the
write shape:

```ts
await database.mutations.tasks.insert({
  id: 'task-1',
  title: 'Ship browser client',
  completed: 0,
  user_id: actorId,
  project_id: projectId ?? null,
});

await database.mutations.tasks.update('task-1', { completed: 1 });
```

For Yjs-backed fields, send `__yjs` envelopes or use the generic CRDT field
runtime APIs. Do not set `title_yjs_state` in app mutations:

```ts
await database.mutations.tasks.update('task-1', {
  __yjs: { title: updateEnvelope },
});
```

Diagnostics are available from the runtime client:

```ts
const snapshot = await database.client.diagnosticSnapshot();
for (const event of snapshot.recentDiagnostics) {
  console.debug(event.code, event.message);
}
```

## Swift

The generated native API exposes the same low-level host contract plus typed
query, mutation, CRDT, live-query, and diagnostic helpers:

```swift
let tasks = try TaskQuery
    .select()
    .filter(TaskQuery.userId.eq(actorId))
    .orderBy(TaskQuery.serverVersion.desc())
    .fetch(on: client)

let commandId = try client.queuedMutations.tasks.insert(NewTask(
    id: "task-ios",
    title: "Ship iOS client",
    completed: 0,
    userId: actorId,
    projectId: projectId
))

let receipt = try client.applyTaskTitleText(
    rowId: "task-ios",
    nextText: "Edited from Swift"
)

let diagnostics = try client.diagnosticSnapshot()
```

Live queries refresh from native events:

```swift
let live = TaskQuery
    .select()
    .filter(TaskQuery.userId.eq(actorId))
    .liveQuery(id: "tasks-main", label: "Tasks")

let initialRows = try live.start(on: client)
let nextRows = try live.refreshIfChanged(event: nativeEvent, on: client)
```

```kotlin
val tasks = TaskQuery
    .select()

    .orderBy(TaskQuery.serverVersion.desc())

val commandId = client.queuedMutations.tasks.insert(

    ),
)

val receipt = client.applyTaskTitleText(
    rowId = "task-android",
    nextText = "Edited from Kotlin",
)

val diagnostics = client.diagnosticSnapshot()
```

Live queries also refresh from native events:

```kotlin
val live = TaskQuery
    .select()

    .liveQuery(id = "tasks-main", label = "Tasks")

val initialRows = live.start(client)
val nextRows = live.refreshIfChanged(nativeEvent, client)
```

## Event Routing

Use generated changed-row helpers to route UI updates from generic runtime
events:

```rust
for task in syncular::task_changed_rows(&event.changed_rows) {
    if task.crdt.title_yjs_state {
        refresh_active_editor(task.row_id());
    }
    if task.is_delete {
        remove_from_sidebar(task.row_id());
    }
}
```

This keeps Syncular generic while giving apps enough typed information to avoid
full bootstrap refreshes for normal row, field, CRDT, conflict, and live-query
changes.
