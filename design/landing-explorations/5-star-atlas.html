All your data, drawn to a single point of truth.

Legend of Symbols

What each mark on the chart denotes.

Scopes as code

Authorization drawn in your own hand — scope functions decide exactly which rows each client may observe. Nothing escapes validation.

Offline outbox

Drift out of contact and keep writing. The outbox replays your commits, in order, the moment you re-enter range.

Honest conflicts

When two observations disagree, we chart both. Conflicts surface to your code — never silently merged behind the map.

Realtime over WebSocket

Commits propagate across the wire at 0.2 ms p95 — light-speed, as far as your UI is concerned.

CRDT columns

For fields that must merge — counters, sets, collaborative text — opt columns into CRDT semantics, per column.

Optional E2EE & blobs

Seal payloads end-to-end when the survey is private; attach blobs alongside rows and sync them through the same log.

Windowed sync

Point the telescope at the sky you need: sync a bounded window of rows per scope, so a decade of history never weighs down a first launch.

Two Cores · One Physics

The protocol is written down like a law of nature, and two independent cores obey it: a TypeScript core for the web and a Rust core for every native platform via C FFI. Both are conformance-locked against the same 74 scenarios, so a commit behaves identically whether it was observed from a browser or a phone in a tunnel.

  • ✦ One written protocol — the fixed stars every core is measured against
  • ✦ 74 conformance scenarios, run against both cores on every change
  • ✦ Rust core reaches Swift, Kotlin, Flutter, React Native & Tauri via C FFI
