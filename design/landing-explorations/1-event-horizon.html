What holds the orbit

Scopes as code
Authorization is written into the sync layer itself. Each client sees exactly the slice of the log its scopes allow — nothing more crosses the horizon.
Offline outbox
Drift out of contact and keep writing. Local commits queue in the outbox and replay, in order, the moment you re-enter range.
Honest conflicts
Conflicting writes are surfaced, never silently merged. You decide the resolution; the log records it.
Realtime propagation
Committed changes fan out over WebSocket at sub-millisecond p95. Every orbiting client feels the pull instantly.
CRDT columns
Opt individual columns into conflict-free merge semantics where last-writer-wins isn’t enough.
Blobs & E2EE
Binary attachments ride the same log, with optional end-to-end encryption when the server shouldn’t see inside.
Windowed sync
Sync only the window of data a device needs. Small clients stay small; history stays on the server.
