# syncular quickstart

The ≤5-minute path: a schema, a ~30-line server, and two client cores that
converge — all in a terminal, no browser. This is the runnable source the
[docs quickstart](../../apps/docs/pages/quickstart.md) walks through; every
snippet there is extracted from these files, and `quickstart.test.ts` runs
the whole thing end to end in the normal `bun test` sweep so it cannot rot.

## Copy it

This directory is a self-contained starting point. Copy it out of the tree
and wire it into your own app:

```sh
cp -r examples/quickstart my-app && cd my-app
```

## Layout

| File | What it is |
|---|---|
| `syncular.json` + `migrations/` | The schema manifest and SQL — typegen's input |
| `src/syncular.generated.ts` | Generated by `syncular generate` (committed) |
| `src/server.ts` | The whole sync backend in one Bun process |
| `src/make-client.ts` | Constructs one `SyncClient` on bun:sqlite + fetch |
| `src/clients.ts` | Two clients, one server, terminal-visible convergence |
| `src/quickstart.test.ts` | The end-to-end smoke test |

## Run it

```sh
bun run generate          # syncular.json + migrations → src/syncular.generated.ts
bun run server            # http://localhost:8787
bun run clients           # in a second terminal — prints "✓ converged"
```
