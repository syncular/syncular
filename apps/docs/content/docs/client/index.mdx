# Client

This section covers the canonical Syncular client runtime and its host
bindings. The Rust runtime owns SQLite, sync state, outbox, conflicts, blobs,
CRDT persistence, encryption, and native bindings.

The current public shape is:

- Rust reads use Diesel query-builder semantics.
- Browser TypeScript reads use Kysely over a Rust-owned SQLite Worker.
- Swift and Kotlin use generated small query DSLs over generic native bindings.
- Writes go through generated Syncular mutations, not raw app-table SQL.
- App-specific code is generated from migrations plus a typed app contract that
  produces `generated/syncular.codegen.json` as the Rust-codegen handoff.

## What stays generic

Low-level runtime and binding APIs are schema-agnostic. They expose SQLite,
sync, events, blobs, CRDT fields, encryption, conflict resolution, and worker
queues without knowing app table names.

## What gets generated

Generated app code provides the type-safe table layer:

- table metadata and migration-derived schemas
- Rust Diesel row types and Syncular mutation builders
- TypeScript Kysely DB types, mutation helpers, subscriptions, and row deltas
- Swift/Kotlin rows, query DSLs, mutations, subscriptions, CRDT helpers, and
  row-delta helpers

This keeps the framework reusable while still giving app code concrete types.
