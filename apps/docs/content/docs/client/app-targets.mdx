# App Targets

Syncular's client path is Rust-first. This section points to the browser,
native, and server runtime guides that matter for the current product shape.

> Start here if you are unsure which runtime to use. If you already chose a
> client target, follow the [Client quick start](/docs/client/quick-start)
> and then come back for runtime caveats.

## Quick recommendations

- **Web:** `@syncular/client` for the Rust-owned browser worker and generated Kysely types.
- **React:** `@syncular/react` over a browser, Tauri, or React Native client.
- **Native apps:** generated Rust, Swift, Kotlin/Android, and JVM clients over the native runtime.
- **Electron / Tauri / React Native:** use the platform bridge package over a Rust/native host.
- **Bun / Node:** run server code with the normal Kysely server dialects; use the Rust SDK for local app storage.
- **Cloudflare:** run the server with `@syncular/server-cloudflare`; browser/native clients stay on the Rust client path.
