# Swift

Swift apps use generated app helpers over the schema-agnostic BoltFFI native
binding. The generated layer gives Swift code table-specific query builders,
rows, mutations, subscriptions, CRDT helpers, and row-delta helpers without
exposing raw synced-table writes.

## Open the native runtime

```swift
let config = SyncularBoltClientConfig(
    dbPath: dbPath,
    baseUrl: "https://api.example.com/sync",
    clientId: "ios-device-1",
    actorId: "user-1",
    projectId: "project-1",
    appSchemaJson: syncularNativeGeneratedAppSchemaJson,
    autoSyncLocalWrites: false
)

let client = try SyncularBoltClient(openAsync: config)
try client.finishOpenTimeout(timeoutMs: 5_000)
try client.startEventStream(capacity: 256)
try client.setAuthHeadersJson(headersJson: #"{"authorization":"Bearer token"}"#)
```

## Query

Generated query DSLs stay close to normal query-builder semantics:

```swift
let rows = try TaskQuery
    .select()
    .filter(TaskQuery.userId.eq("user-1"))
    .orderBy(TaskQuery.serverVersion.desc())
    .limit(20)
    .fetch(on: client)
```

## Mutate

Generated mutation helpers call the native Syncular mutation path:

```swift
let commitId = try client.mutations.tasks.insert(NewTask(
    id: UUID().uuidString,
    title: "Swift task",
    completed: 0,
    userId: "user-1",
    projectId: "project-1"
))

try client.mutations.tasks.update(
    rowId: taskId,
    patch: TaskPatch(completed: 1),
    baseVersion: 11
)
```

For UI commands that should not block the main thread, use queued mutations and
listen for the matching command event through the native event stream.

## Events

Swift hosts can consume decoded native events directly:

```swift
extension SyncularBoltClient: SyncularNativeEventJsonSource {}

for try await event in syncularNativeEventStream(from: client) {
    await viewModel.apply(syncularEvent: event)
}
```

Route row/field metadata with generated helpers:

```swift
for task in taskChangedRows(in: event) {
    if task.changed.title {
        refreshTask(task.rowId)
    }
}
```

Use [Native lifecycle](/docs/client/native) for worker backpressure, realtime,
presence handles, CRDT field persistence, event overflow, and shutdown policy.
