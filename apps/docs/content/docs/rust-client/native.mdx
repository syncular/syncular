# Native Apps

Native apps use the same Rust runtime foundation as browser apps, but access it
through the Rust SDK or generated BoltFFI bindings.

## Rust

Rust is the canonical native SDK surface. Reads use Diesel, and writes use
generated Syncular mutations:

```rust
let rows = client.read(
    schema::tasks::dsl::tasks
        .filter(schema::tasks::dsl::user_id.eq("user-rust"))
        .order(schema::tasks::dsl::server_version.desc())
        .select(diesel_tables::TaskRow::as_select()),
)?;

client
    .mutations()
    .tasks()
    .update(syncular::TaskPatch::new(task_id).completed(1))?;
```

## Swift and Kotlin

Swift and Kotlin get generated app helpers over the schema-agnostic native
binding:

- query DSLs for typed read SQL
- generated mutation builders
- generated subscription helpers
- typed CRDT field helpers
- typed row-delta helpers

Swift:

```swift
let rows = try TaskQuery
    .select()
    .filter(TaskQuery.userId.eq("user-rust"))
    .orderBy(TaskQuery.serverVersion.desc())
    .fetch(on: client)

for task in taskChangedRows(in: event) {
    if task.changed.title { refreshTask(task.rowId) }
}
```

Kotlin:

```kotlin
val rows = TaskQuery
    .select()
    .filter(TaskQuery.userId.eq("user-rust"))
    .orderBy(TaskQuery.serverVersion.desc())
    .fetch(client)

for (task in taskChangedRows(event)) {
    if (task.changed.title) refreshTask(task.rowId)
}
```

## UI lifecycle

UI shells should keep one hot native runtime per app/session. Prefer queued
APIs for work that can block:

- `enqueueMutationJson`
- `enqueueSyncNow`
- `enqueueSyncWebsocket`
- queued CRDT field updates and compaction
- queued blob upload/cache/compaction work

Start the native event stream once, read `nextEventJson()` from a background
task, and route updates by `eventSeq`, `commandId`, `changedRows`, and query
ids. The native worker owns backpressure for queued local writes and CRDT
bursts. During teardown, close the event stream and then call `shutdown()`.

Swift:

```swift
try client.startEventStream(capacity: 256)

Task.detached {
    while let eventJson = try client.nextEventJson() {
        let event = try syncularDecodeNativeEvent(eventJson)
        await viewModel.apply(syncularEvent: event)
    }
}
```

Kotlin:

```kotlin
client.startEventStream(256uL)

scope.launch(Dispatchers.IO) {
    while (true) {
        val eventJson = client.nextEventJson() ?: break
        val event = syncularDecodeNativeEvent(eventJson)
        withContext(Dispatchers.Main) {
            viewModel.applySyncularEvent(event)
        }
    }
}
```

## Realtime and sync

HTTP sync is the normal path. WebSocket sync is available for native hosts
through `enqueueSyncWebsocket` when the server route supports Syncular's
WebSocket push transport. Realtime notifications remain wake-ups; persisted
row state still comes from sync.
