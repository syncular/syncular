# Packaging

Native packages are generated from the current Rust runtime, headers, and JNI
glue in one pass. Do not treat checked-in generated binding files or binary
artifacts as release truth.

The current native FFI ABI version is `2`. Generated app clients assert this
against the runtime manifest before opening data, so regenerate app clients after
updating the runtime or native bindings.

## Local release check

Run from the repo root:

```bash
bun run rust:native:release-check
```

This builds:

- Apple `Syncular.xcframework`, SwiftPM package, zip, and checksums
- Android JNI libraries for `arm64-v8a` and `x86_64`
- Android AAR in a local Maven repository
- current-host JVM native package
- Linux x86_64 JVM native package
- generated Swift/Kotlin/JVM native smoke tests

The native smoke covers the generated event-stream APIs
(`startEventStream`, `nextEventJson`, and `closeEventStream`) in addition to
sync, auth refresh, realtime events, conflicts, blobs, and encrypted fields.

Artifacts are written to `.context/native-packages` by default. Override with:

```bash
SYNCULAR_NATIVE_PACKAGE_OUT=/path/to/out bun run rust:native:release-check
```

## Targeted commands

```bash
bun run rust:native:package:apple
bun run rust:native:package:android
bun run rust:native:package:java
bun run rust:native:package:java:linux
bun run rust:native:package:java:windows
```

Windows JVM packaging needs a Windows host or runner with the current BoltFFI
backend. The repository has a Windows packaging command and CI job shape, but
macOS cannot cross-package that target with BoltFFI `0.24.1`.

## Apple

Apple packaging emits:

- `Syncular.xcframework`
- `Syncular.xcframework.zip`
- `Syncular.xcframework.zip.swift-checksum`
- `Syncular.xcframework.zip.sha256`
- `Package.swift`
- generated Swift BoltFFI wrappers and headers

For SwiftPM binary releases, upload the zip and use the Swift checksum file as
the `binaryTarget` checksum.

## Android

Android packaging publishes the schema-agnostic low-level binding as:

```text
dev.syncular:syncular-android:<runtime-version>
```

App-specific generated Kotlin code stays in the consuming app and depends on
that AAR.

Local consumer setup:

```kotlin
dependencyResolutionManagement {
    repositories {
        google()
        mavenCentral()
        maven {
            url = uri("/path/to/syncular/.context/native-packages/android-maven/repository")
        }
    }
}
```

```kotlin
dependencies {
    implementation("dev.syncular:syncular-android:0.1.0")
}
```

For publication signing, provide `SYNCULAR_MAVEN_SIGNING_KEY` and
`SYNCULAR_MAVEN_SIGNING_PASSWORD` to the packaging command.

## JVM

The JVM package includes Java wrappers and native libraries under
`java/native/<host>`. The current host is packaged by default. Linux x86_64
cross-packaging from macOS requires `zig` and the Rust
`x86_64-unknown-linux-gnu` target.
