# Fleet

Fleet is the client-centric operational view. Use it to identify which clients are behind, stale, disconnected, or reporting unhealthy Rust runtime diagnostics.

## Key concepts

- **Head**: latest commit sequence on the server.
- **Cursor**: last commit sequence acknowledged by a client.
- **Lag**: `head - cursor` for each client.
- **Activity state**: online, syncing, or stale/offline.
- **Runtime freshness**: age of the latest retained redacted Rust client diagnostic snapshot.
- **Runtime health**: highest diagnostic severity reported by the latest retained snapshot.

## What to check

1. **Max lag** trend over time.
2. **Which clients are behind** and by how much.
3. **Connection mode** (`realtime` vs `polling`) and path (`direct` vs `relay`).
4. **Runtime freshness and health** for stale snapshots, warnings, and errors.
5. **Effective scopes** for clients that appear missing data.

## Runtime diagnostics

Rust browser/native clients can post redacted diagnostic snapshots to
`POST /console/client-diagnostics`. Console stores normalized, size-bounded
records in `sync_client_diagnostic_snapshots` and uses them to show:

- latest runtime package/crate/schema evidence;
- lifecycle, realtime, queue, conflict, blob-upload, and timing summaries;
- stable diagnostic code summaries and highest severity;
- retained snapshot history in the client drilldown.

Snapshots reject common sensitive-key fields such as tokens, passwords, secrets,
and plaintext payloads. The Console UI labels the retained evidence as redacted
runtime state; it is investigation context, not an authorization source.

Starter/browser preview smokes can also post the raw
`browser-preview-failure.json` artifact, or a wrapper that adds
`clientId`/`actorId`/`partitionId`, to
`POST /console/client-diagnostics/browser-preview-failure`. Console normalizes
that artifact into a `browser.preview_failure` diagnostic record with safe
asset metrics, deployment-preflight facts, support-bundle counts, lifecycle
timings, and timeline counters. The bounded page `textExcerpt` from the
artifact is not retained in the normalized Console record.

Cloudflare framework/runtime smokes can post the raw
`cloudflare-runtime-failure.json` artifact, or the same identity wrapper, to
`POST /console/client-diagnostics/cloudflare-runtime-failure`. Console stores
it as `cloudflare.runtime_failure` with the failed route, sync/blob/WebSocket
route bases, exit status, bounded output excerpt, and R2 blob route
timing/byte metrics.

## Evicting a client

Use **Evict** when a client has irrecoverable cursor/state issues.

Effect:

- deletes that client cursor record
- forces bootstrap on next sync cycle
- creates an operation-audit entry

Use partition filters before eviction in multi-tenant setups.
