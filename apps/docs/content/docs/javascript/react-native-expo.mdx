# React Native / Expo

React Native apps use `@syncular/client-react-native` on the JavaScript side
and a native Syncular module on the host side. JavaScript gets typed Kysely
reads, generated mutations, events, presence, conflicts, and React hooks. The
native module owns SQLite, sync, outbox, realtime, blobs, CRDT fields,
encryption, auth leases, and lifecycle.

Expo apps use the same React Native bridge once the native module is available
in the app build. This is a native runtime path, so it is not an Expo Go-only
integration.

## Install

```bash
npm install @syncular/client @syncular/client-react-native @syncular/react kysely
```

## Create the bridge client

```ts
import { createSyncularNativeClient } from '@syncular/client-react-native';
import type { AppDb } from './generated/syncular.browser';

const syncular = await createSyncularNativeClient<AppDb>({
  module: SyncularNativeModule,
});
```

The module implements the small bridge contract: `executeSql`,
`applyMutationsCommit`, lifecycle methods, subscriptions, event listeners,
presence, conflicts, auth leases, and diagnostics.

## React usage

```tsx
import {
  createSyncularNativeClient,
  createSyncularNativeReact,
} from '@syncular/client-react-native';

const syncular = await createSyncularNativeClient<AppDb>({
  module: SyncularNativeModule,
});

const { SyncProvider, useSyncQuery, useMutations, useSyncConnection } =
  createSyncularNativeReact<AppDb>();

export function App() {
  return (
    <SyncProvider client={syncular}>
      <Tasks />
    </SyncProvider>
  );
}
```

Queries stay Kysely-shaped:

```ts
const tasks = await syncular.db
  .selectFrom('tasks')
  .selectAll()
  .where('user_id', '=', actorId)
  .execute();
```

Writes stay on the generated Syncular mutation path:

```ts
await syncular.mutations.tasks.update(taskId, { completed: 1 });
await syncular.leasedMutations.tasks.update(taskId, { title: 'Offline edit' });
```

## Lifecycle

Wire app foreground/background transitions to the native runtime:

```ts
await syncular.resumeFromBackground();
await syncular.start();
await syncular.stop();
```

The bridge preserves `rowsChanged.changedRows` row/field metadata. Use that
metadata to refresh active documents, lists, and read models without falling
back to table-level reloads.
