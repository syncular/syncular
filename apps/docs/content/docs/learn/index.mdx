# Learn

Use this section to understand how Syncular works before diving into
implementation details. These pages describe the shared product model: a
server-authoritative commit log, scoped subscriptions, generated app contracts,
and a Rust-owned client runtime behind JavaScript, Rust, and native host APIs.

> **You are here:** Learn (mental model and semantics).
>
> **If you're implementing:** continue with [JavaScript](/clients/javascript) or
> [Rust](/clients/rust), then [Server](/server) and
> [Features](/features).

Recommended reading order:

1. [Architecture](/learn/architecture)
2. [First Sync](/learn/first-sync)
3. [Scopes](/learn/scopes)
4. [Subscriptions](/learn/subscriptions)
5. [Conflict Resolution](/learn/conflict-resolution)
