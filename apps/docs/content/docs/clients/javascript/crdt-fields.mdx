# CRDT Fields

Syncular treats CRDT-backed document fields as runtime primitives. JavaScript
apps can attach TipTap, ProseMirror, Excalidraw, or other editor adapters on
top, but the runtime owns durable update storage, merge/materialization,
coalescing, compaction, encryption hooks, and sync.

## Configure a field

Declare CRDT fields in `syncular.app.ts`:

```ts
syncedTable({
  table: 'notes',
  serverVersion: 'server_version',
  crdt: {
    content: yjsText({ stateColumn: 'content_yjs_state' }),
  },
});
```

## Apply editor updates

Editor adapters should enqueue updates instead of writing synced table columns
directly:

```ts
await syncular.client.applyCrdtFieldYjsUpdate({
  table: 'notes',
  rowId: noteId,
  field: 'content_yjs_state',
  update,
});
```

For high-frequency editor bursts, use queued APIs so the runtime can coalesce
updates by `(table, rowId, field)` before writing SQLite and the outbox.

## Materialize

Materialize when a view model needs a durable JSON/text projection:

```ts
const materialized = await syncular.client.materializeCrdtField({
  table: 'notes',
  rowId: noteId,
  field: 'content_yjs_state',
});
```

Keep title, preview, outline, and search fields in app-owned read models.
Selection, composition, scroll, and undo stacks stay in the editor layer.

## Compaction

Run compaction through the runtime when update logs become large. Compaction
should preserve the current document, write a checkpoint, and prune redundant
acknowledged updates.
