# Quick Start

## Install

This page uses the TypeScript testkit. For Rust/native generated client tests,
use [Rust Testing](/clients/rust/testing).

```bash
bun add -d @syncular/testkit
```

## Your first test

`createHttpServerFixture` boots real `@syncular/server/hono` routes against an
in-memory database and serves them over `node:http` on a random port. The
testkit also ships a ready-made `tasks` table (DDL + server handler + protocol
builders), so the first test needs no schema of your own:

```ts
import { describe, expect, it } from 'bun:test';
import {
  createHttpServerFixture,
  createProjectScopedTasksHandler,
  createProjectScopedTasksSubscription,
  createProjectScopedTaskUpsertOperation,
  createSyncCombinedRequest,
  ensureProjectScopedTasksTable,
  postSyncCombinedRequest,
} from '@syncular/testkit';

describe('sync quick start', () => {
  it('pushes a row and pulls it back', async () => {
    const server = await createHttpServerFixture({
      serverDialect: 'sqlite',
      createTables: ensureProjectScopedTasksTable,
      handlers: [createProjectScopedTasksHandler()],
      authenticate: async (c) => {
        const actorId = c.req.header('x-actor-id');
        return actorId ? { actorId } : null;
      },
    });

    try {
      const { json } = await postSyncCombinedRequest({
        fetch,
        url: `${server.baseUrl}/sync`,
        actorId: 'u1',
        body: createSyncCombinedRequest({
          clientId: 'client-a',
          push: {
            commits: [
              {
                clientCommitId: 'commit-1',
                schemaVersion: 1,
                operations: [
                  createProjectScopedTaskUpsertOperation({
                    taskId: 'task-1',
                    title: 'First synced task',
                  }),
                ],
              },
            ],
          },
          pull: {
            schemaVersion: 1,
            limitCommits: 50,
            subscriptions: [
              createProjectScopedTasksSubscription({ userId: 'u1' }),
            ],
          },
        }),
      });

      expect(json.push?.commits[0]?.status).toBe('applied');
      expect(json.pull?.subscriptions[0]?.status).toBe('active');

      // The write really landed in the server database.
      const row = await server.db
        .selectFrom('tasks')
        .select(['id', 'title', 'user_id'])
        .where('id', '=', 'task-1')
        .executeTakeFirstOrThrow();
      expect(row.user_id).toBe('u1');
    } finally {
      await server.destroy();
    }
  });
});
```

Everything in this test is real: the Hono routes, the SQLite database behind
`server.db`, and the combined `POST /sync` request a production client sends.
`postSyncCombinedRequest` validates the response against the protocol schemas
before returning it.

## Swap in your own table

Replace `createTables` and `handlers` with your own DDL and
[table handler](/server/table-handlers), and build operations with the generic
builders (`createSyncUpsertOperation`, `createSyncSubscription`) instead of the
task-specific ones. [Testing Strategy — layer 1](/testing/strategy#layer-1--handler-tests)
shows the full per-table checklist: happy-path apply, scope revocation, and
handler-specific rejections.

## Postgres semantics

Run the same fixture against PGLite when you deploy on Postgres:

```ts
const server = await createHttpServerFixture({
  serverDialect: 'pglite',
  // everything else unchanged
  createTables: ensureProjectScopedTasksTable,
  handlers: [createProjectScopedTasksHandler()],
  authenticate: async (c) => {
    const actorId = c.req.header('x-actor-id');
    return actorId ? { actorId } : null;
  },
});
```

## Testing through the real client

For two-device convergence and offline/outbox tests, open your generated app
client from `@syncular/client` against `server.baseUrl` — that exercises the
real WASM SQLite runtime, outbox, and cursor tracking. See
[Testing Strategy — layers 2 and 3](/testing/strategy#layer-2--two-client-convergence)
for the worked pattern.
