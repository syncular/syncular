# Resource Lifecycle (`using`)

`@syncular/testkit` supports two cleanup styles built on one primitive,
`AsyncDisposableResource<T>` (`{ value, dispose, [Symbol.asyncDispose] }`):

- callback helpers (`with*`) that work in every environment
- resource helpers (`create*Resource` / `open*Resource`) for `await using`
  style cleanup

Both styles are runner-agnostic and disposal is idempotent — calling
`dispose()` twice is safe.

## Wrap any fixture

`createAsyncDisposableResource(value, dispose)` turns anything with a teardown
into a resource. The most common case is wrapping `createHttpServerFixture`:

```ts
import {
  createAsyncDisposableResource,
  createHttpServerFixture,
  createProjectScopedTasksHandler,
  ensureProjectScopedTasksTable,
} from '@syncular/testkit';

async function openTasksServer() {
  const server = await createHttpServerFixture({
    serverDialect: 'sqlite',
    createTables: ensureProjectScopedTasksTable,
    handlers: [createProjectScopedTasksHandler()],
    authenticate: async (c) => {
      const actorId = c.req.header('x-actor-id');
      return actorId ? { actorId } : null;
    },
  });

  return createAsyncDisposableResource(server, () => server.destroy());
}
```

## `using` style (optional)

```ts
await using serverRes = await openTasksServer();
const server = serverRes.value;

// ...test body; server.destroy() runs automatically on scope exit
```

## Callback style (universal)

```ts
import { withAsyncDisposableFactory } from '@syncular/testkit';

await withAsyncDisposableFactory(openTasksServer, async (server) => {
  // ...test body; cleanup runs in finally
});
```

`withAsyncDisposableResource(resource, run)` does the same for a resource you
already created.

## Built-in resource helpers

| Resource | `await using` | Callback |
|---|---|---|
| Node HTTP server for a Hono app | `createNodeHonoServerResource(app, options)` | `withNodeHonoServer(app, run, options)` |
| Realtime WebSocket | `openRealtimeWsResource({ ..., waitForOpen: true })` | `withRealtimeWs(options, run)` |
| Child process (spawned server) | `createChildProcessResource(child)` | `withChildProcess(child, run)` |
| Existing `node:http` server | `createNodeServerResource(server)` | — |
| Bridge harness | `createClientBridgeHarness(options)` (returns a resource directly) | — |

```ts
import { openRealtimeWsResource } from '@syncular/testkit';

await using wsRes = await openRealtimeWsResource({
  baseUrl: 'http://localhost:3000',
  clientId: 'client-b',
  actorId: 'u1',
  waitForOpen: true,
});
const ws = wsRes.value;
```

## Keep non-resource helpers simple

Do not use `using` for pure helpers:

- protocol builders/parsers
- deterministic id/clock factories
- pull-inspection helpers and fault wrappers

Those remain plain functions with nothing to dispose.
