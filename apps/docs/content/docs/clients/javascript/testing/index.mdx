# Testing

`@syncular/testkit` lets TypeScript tests run against real pieces instead of
mocks: real `@syncular/server/hono` routes served over `node:http`, real
in-memory SQLite/Postgres (PGLite) databases, real protocol payloads validated
by the shared schemas, and transport wrappers for deterministic fault
injection.

```bash
bun add -d @syncular/testkit
```

The API is test-runner agnostic — Bun, Vitest, Jest, and other runners work
without changing testkit code. For Rust/native generated clients, use
[Rust Testing](/clients/rust/testing).

Looking for the what-to-verify-before-shipping guide (the four test layers,
each with a runnable example)? Start at
[Testing Strategy](/testing/strategy). This section is the API-by-API guide
to the testkit itself.

## API groups

These groups cover the TypeScript `@syncular/testkit` package:

| Group | Main exports |
|---|---|
| HTTP server fixture | `createHttpServerFixture`, `createNodeHonoServer`, `closeNodeServer` |
| Reference tasks schema | `createProjectScopedTasksHandler`, `ensureProjectScopedTasksTable`, `createProjectScopedTasksSubscription`, `createProjectScopedTaskUpsertOperation` |
| Protocol builders | `createSyncUpsertOperation`, `createSyncDeleteOperation`, `createSyncSubscription`, `createSyncPushRequest`, `createSyncPullRequest`, `createSyncCombinedRequest` |
| HTTP protocol calls | `postSyncCombinedRequest`, `postSyncPushRequest`, `postSyncPullRequest`, `postJsonWithActor`, `createJsonActorHeaders` |
| Response parsing | `parseSyncCombinedResponse`, `parseSyncPushResponse`, `parseSyncPullResponse`, `readSyncCombinedResponse`, `readSyncPushResponse`, `readSyncPullResponse` |
| Pull inspection | `subscriptionChanges`, `findSubscriptionChange`, `subscriptionChangeRow` |
| Faults + recording | `withFaults`, `createMockTransport`, `withRecording` |
| Deterministic IDs/clocks | `createIdFactory`, `createCommitIdFactory`, `createFakeClock` |
| Realtime WebSocket | `createRealtimeWsUrl`, `openRealtimeWs`, `waitForWsOpen`, `waitForWsMessage`, `waitForWsJsonMessage`, `parseWsJsonMessage`, `closeWsSafe` |
| Resource lifecycle | `createAsyncDisposableResource`, `withAsyncDisposableFactory`, `createNodeHonoServerResource`, `withNodeHonoServer`, `openRealtimeWsResource`, `withRealtimeWs`, `createChildProcessResource`, `withChildProcess` |
| Child processes | `waitForJsonPortFromStdout`, `stopChildProcess` |
| Bridge harness | `createClientBridgeHarness` (in-process `SyncularBridge` for Tauri/React Native shell tests) |
| Audit assertions | `assertAuditChangeRedacted`, `assertAuditJsonExcludes`, `assertAuditDebugExportRedacted` |

Tests that exercise the full client runtime (two-device convergence, offline
outbox drain) use your generated app client from `@syncular/client` pointed at
a testkit server fixture — see
[Testing Strategy, layers 2 and 3](/testing/strategy#layer-2--two-client-convergence)
for the pattern.
