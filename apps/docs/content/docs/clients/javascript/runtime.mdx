# Runtime

JavaScript app code owns typed Kysely reads, generated mutations, host UI, and
framework adapters. The Syncular runtime owns SQLite, migrations, outbox,
conflicts, blobs, CRDT fields, encryption, realtime wakeups, retry behavior,
and row/field events.

## Browser worker

Browser apps run SQLite and sync inside a dedicated Rust WASM Worker. App code
talks to the worker through the generated client:

```ts
const syncular = await createSyncularAppDatabase({
  config: {
    baseUrl: '/sync',
    actorId,
    clientId,
    fileName: 'app.sqlite',
  },
});
```

Use [Browser](/clients/javascript/browser) for worker setup details.

## Native host bridges

React Native/Expo, Tauri, and Electron should keep the runtime outside the
renderer/UI thread. The JavaScript side receives a Kysely-compatible query
surface, generated mutations, lifecycle commands, and event subscriptions.

## Lifecycle

Keep one hot runtime per user session. Start it when the app can sync, pause or
stop it on background transitions when the host requires it, and close it when
the user signs out or the local replica is discarded.

Queued APIs should be used for work that can block the UI:

- local mutations;
- sync commands;
- CRDT field bursts;
- blob upload/download/cache work.

## Events

Runtime events are the contract for UI refresh and app chrome. They include:

- `rowsChanged`
- `queriesChanged`
- `outboxChanged`
- `conflictsChanged`
- `presenceChanged`
- `blobUploadCompleted`
- `blobUploadFailed`
- overflow or resync-required events

Use [Live Updates](/clients/javascript/live-updates) for event routing.
