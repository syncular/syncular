# React

React is a host layer over an existing Syncular client. It does not own sync or
SQLite. The underlying client can be the browser Worker client, the Tauri
bridge, or the React Native bridge.

## Install

```bash
npm install @syncular/client @syncular/react kysely
```

For browser apps, create the client with `@syncular/client` and pass it into
the generated React surface:

```tsx
import { createSyncularClient } from '@syncular/client';
import { createSyncularReact } from '@syncular/react';
import type { AppDb } from './generated/syncular.browser';

const syncular = await createSyncularClient<AppDb>({
  config: {
    baseUrl: '/sync',
    actorId: 'user-1',
    clientId: 'device-1',
  },
  realtime: true,
});

const {
  SyncProvider,
  useBlobUploadQueue,
  useConflictStats,
  useLeasedMutations,
  useMutations,
  useOutboxStats,
  usePresenceWithJoin,
  useRowsChanged,
  useSyncConnection,
  useSyncQuery,
} = createSyncularReact<AppDb>();

export function App() {
  return (
    <SyncProvider client={syncular}>
      <Tasks />
    </SyncProvider>
  );
}
```

Platform clients provide matching React factories:

```ts
import { createSyncularNativeReact } from '@syncular/client-react-native';
import { createSyncularTauriReact } from '@syncular/client-tauri';
```

## Query

Reads stay query-builder based. `useSyncQuery` passes a typed Kysely context and
refreshes when relevant row/query events arrive:

```tsx
function Tasks() {
  const { data: tasks = [], isLoading } = useSyncQuery(
    ({ selectFrom }) =>
      selectFrom('tasks')
        .select(['id', 'title', 'completed'])
        .where('user_id', '=', 'user-1')
        .orderBy('server_version', 'desc'),
    {
      watchTables: ['tasks'],
    }
  );

  if (isLoading) return null;
  return tasks.map((task) => <TaskRow key={task.id} task={task} />);
}
```

## Mutate

Writes go through generated Syncular mutation helpers. The hook preserves the
same mutation surface as the underlying client:

```tsx
function TaskRow({ task }: { task: TaskRow }) {
  const mutations = useMutations({ sync: 'background' });

  return (
    <button
      onClick={() =>
        mutations.tasks.update(task.id, {
          completed: task.completed ? 0 : 1,
        })
      }
    >
      {task.title}
    </button>
  );
}
```

Use leased mutations for offline-auth windows:

```ts
const leased = useLeasedMutations();
await leased.tasks.update(taskId, { title: 'Offline edit' });
```

## App chrome

The React package exposes hooks for status, connection state, rows changed,
outbox stats, conflicts, presence, blobs, and blob upload queues:

```tsx
const connection = useSyncConnection();
const outbox = useOutboxStats();
const conflicts = useConflictStats();
const uploads = useBlobUploadQueue();

const editors = usePresenceWithJoin('user:user-1', {
  metadata: { view: 'tasks' },
});
```

For lower-level row routing, subscribe to row/field metadata:

```tsx
useRowsChanged({
  tables: ['tasks'],
  enabled: true,
});
```

Generated row-delta helpers from the app client turn generic runtime events
into table-specific refresh decisions.
