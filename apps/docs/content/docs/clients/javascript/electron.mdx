# Electron

Electron apps should keep the Syncular runtime in the main process, a native
host, or another dedicated runtime boundary. Renderer code gets typed Kysely
reads, generated mutation helpers, and event subscriptions through a small
preload bridge.

## Recommended shape

- Keep SQLite, sync, outbox, realtime, blobs, CRDT fields, encryption, and auth
  refresh out of the renderer.
- Expose only generated reads, generated mutations, lifecycle commands, blob
  APIs, and event subscriptions through preload IPC.
- Reject raw app-table inserts and updates from renderer SQL execution.

## Renderer usage

```ts
const rows = await syncular.db
  .selectFrom('tasks')
  .selectAll()
  .where('user_id', '=', actorId)
  .execute();

await syncular.mutations.tasks.insert({
  id: crypto.randomUUID(),
  title: 'Electron task',
  user_id: actorId,
});
```
