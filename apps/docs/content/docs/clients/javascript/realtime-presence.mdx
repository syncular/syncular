# Realtime And Presence

Realtime is a wakeup path. HTTP sync remains the authoritative catchup path for
missed commits, reconnects, auth refresh, and recovery after event overflow.

## Start realtime

Realtime is on by default for remote databases — opening one registers
subscriptions, runs the initial sync, and connects the websocket:

```ts
const syncular = await createSyncularDatabase<AppDb>({
  config,
  subscriptions,
});
```

To defer startup, open with `lifecycle: { autoStart: false }` and start
explicitly later:

```ts
await syncular.setSubscriptions(subscriptions);
await syncular.start();
```

## Presence

Join presence after subscriptions and initial sync are established:

```ts
const unsubscribe = syncular.presence.onChange((event) => {
  renderPresence(event.scopeKey, event.presence);
});

syncular.presence.join('project:project-1', {
  view: 'task-list',
});

syncular.presence.updateMetadata('project:project-1', {
  view: 'task-detail',
  taskId,
});

syncular.presence.leave('project:project-1');
unsubscribe();
```

## Recovery model

Use this rule in app code:

- websocket event: wake sync and refresh from local SQLite after sync
- websocket reconnect: run catchup sync
- event overflow: resubscribe and refresh live queries
- auth failure: refresh token or show a session-expired state

This keeps realtime fast without making it responsible for durable state.
