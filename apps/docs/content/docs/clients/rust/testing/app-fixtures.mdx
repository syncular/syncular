# App Fixtures

`syncular-testkit` has several fixture levels. Use the smallest level that
still exercises the behavior you care about.

## Local generated client

Use `open_app_client(...)` when you want a real client and a scripted transport:

```rust
use syncular_testkit::open_app_client;

let mut fixture = open_app_client(my_app::generated::app_schema())?;

fixture.transport.push_http_response(default_combined_response());
fixture.client.sync_http()?;
```

This is useful for asserting exact requests, auth headers, retry status, and
error handling.

## In-memory SQLite

Use `open_app_client_in_memory(...)` for fast client tests that do not need to
reopen the database:

```rust
let mut fixture = syncular_testkit::open_app_client_in_memory(
    my_app::generated::app_schema(),
)?;
```

Use file-backed temp DBs when testing migrations, reopen behavior, blob cache
paths, or persisted command/outbox state.

## Stateful server

Use `open_app_client_with_server(...)` when multiple clients should push and
pull through shared server state:

```rust
let server = syncular_testkit::AppTestServer::new(my_app::generated::app_schema());

let writer = syncular_testkit::open_app_client_with_server(
    my_app::generated::app_schema(),
    server.clone(),
    syncular_testkit::AppFixtureOptions {
        client_id: "writer".to_string(),
        ..Default::default()
    },
)?;
```

## Native fixture

Use native fixtures when testing the native facade, event subscriptions, or
binding-facing JSON APIs:

```rust
use syncular_testkit::{
    open_native_client_with_schema, wait_native_event,
};
use syncular_client::native::NativeEventKind;

let mut fixture = open_native_client_with_schema(my_app::generated::app_schema())?;

let command_id = fixture.client.enqueue_sync_now()?;
let event = wait_native_event(
    &fixture.events,
    NativeEventKind::SyncCompleted,
    std::time::Duration::from_secs(5),
);

assert_eq!(event.command_id.as_deref(), Some(command_id.as_str()));
```

## Fixture options

`AppFixtureOptions` controls client identity and scope defaults:

```rust
AppFixtureOptions {
    db_prefix: "my-app-test".to_string(),
    base_url: "http://syncular.test/sync".to_string(),
    client_id: "device-a".to_string(),
    actor_id: "user-rust".to_string(),
    project_id: Some("p0".to_string()),
}
```

Use stable `actor_id`, `client_id`, and scope values in tests so failures are
easy to compare.
