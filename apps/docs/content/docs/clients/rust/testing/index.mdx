# Testing

Use `syncular-testkit` for Rust and native app tests. It is the Rust-side
counterpart to the TypeScript `@syncular/testkit`: avoid mocking Syncular
internals while still keeping tests local, deterministic, and fast.

Use [JavaScript Testing](/docs/clients/javascript/testing) for
TypeScript/server/browser testkit APIs. Use this page when the test opens
generated Rust/native clients or validates native runtime behavior.

## Add the testkit

```toml
[dev-dependencies]
syncular-client = { version = "0.1", default-features = false, features = ["native", "crdt-yjs", "e2ee"] }
syncular-testkit = "0.1"
```

## What the Rust testkit covers

The Rust testkit includes:

- disposable SQLite paths and cleanup
- scripted sync transports and local HTTP sync servers
- a stateful app test server for generated schemas
- blob upload/download test servers
- faulted transports for retry and duplicate delivery cases
- native client fixtures and event waiters
- CRDT field helpers and assertions
- outbox, conflict, blob queue, and cache assertions

Use the testkit for app behavior. Keep app-specific fixtures, assertions, and
domain rows in your app test layer.
