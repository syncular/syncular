# Rust

Rust is the canonical Syncular client runtime. Use this section when your app
code is Rust, or when a native shell embeds Syncular through the Rust runtime.
The Rust path is not an ORM layer over Syncular. Reads stay Diesel
query-builder expressions, while writes go through generated Syncular mutation
namespaces so local rows, outbox commits, conflicts, blobs, CRDT fields, and
encryption remain coherent.

## Runtime split

| Layer | Rust responsibility |
|---|---|
| Generated app client | Diesel schema, row structs, table metadata, mutation namespaces, subscriptions, changed-row helpers |
| `syncular-client` | Developer-facing Rust SDK for typed reads, generated writes, sync, conflicts, and live queries |
| `syncular-runtime` | SQLite, migrations, outbox, protocol, native worker, event streams, blobs, CRDT fields, encryption, realtime |
| `syncular-testkit` | Real SQLite fixtures, in-memory app server, transports, assertions, native event helpers, conformance data |
| Native bindings | BoltFFI/C ABI surfaces used by Swift, Kotlin, JVM, React Native, Tauri, and Electron hosts |

## Recommended path

1. Start with [Quick Start](/docs/clients/rust/quick-start).
2. Read [Generated Client](/docs/clients/rust/generated-client) before
   changing migrations or `syncular.app.ts`.
3. Use [Querying](/docs/clients/rust/querying) and
   [Mutations](/docs/clients/rust/mutations) for day-to-day app code.
4. Use [Runtime](/docs/clients/rust/runtime) for UI apps and host bridges.
5. Use [Native Hosts](/docs/clients/rust/native-hosts) when Swift, Kotlin,
   JVM, React Native, Tauri, or Electron hosts embed the Rust runtime.
6. Add [Testing](/docs/clients/rust/testing) before building app-specific sync
   behavior.

For browser, React, React Native, Tauri, Electron, and TypeScript testing, use
[JavaScript](/docs/clients/javascript).
