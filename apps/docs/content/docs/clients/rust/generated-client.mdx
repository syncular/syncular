# Generated Client

Rust app code should not hand-author table metadata. Generate it from your
SQLite migrations and the `syncular.app.ts` client contract. The output is app
code that belongs in the consuming app repository, not framework code checked
into Syncular itself.

## Inputs

The generator reads two inputs:

- `migrations/**/up.sql`: SQLite table shape for the local replica.
- `generated/syncular.codegen.json`: generated handoff from
  `syncular-typegen codegen-config --app ./syncular.app.ts`.

The TypeScript contract is used at development time even for Rust apps because
it is the current shared schema definition surface. It describes Syncular
semantics that SQL alone cannot express: subscriptions, scopes, server-version
columns, soft deletes, CRDT fields, encrypted fields, and blob reference
columns.

## Outputs

Rust codegen emits:

```text
generated/rust/
  schema.rs         # Diesel table! declarations
  diesel_tables.rs  # Selectable/Queryable row structs and adapters
  migrations.rs     # Embedded migration SQL
  syncular.rs       # table metadata, subscriptions, mutations, changed-row helpers
```

Include those modules from your app crate:

```rust
pub mod generated {
    pub mod schema {
        include!(concat!(env!("CARGO_MANIFEST_DIR"), "/generated/rust/schema.rs"));
    }
    pub mod diesel_tables {
        include!(concat!(env!("CARGO_MANIFEST_DIR"), "/generated/rust/diesel_tables.rs"));
    }
    pub mod migrations {
        include!(concat!(env!("CARGO_MANIFEST_DIR"), "/generated/rust/migrations.rs"));
    }
    pub mod syncular {
        include!(concat!(env!("CARGO_MANIFEST_DIR"), "/generated/rust/syncular.rs"));
    }
}
```

## App schema

Wire generated metadata into the runtime once:

```rust
use syncular_client::app_schema::AppSchema;

pub fn app_schema() -> AppSchema {
    AppSchema {
        app_tables: generated::syncular::APP_TABLES,
        app_table_metadata: generated::syncular::APP_TABLE_METADATA,
        migrations: generated::migrations::MIGRATIONS,
        schema_version: Some(generated::migrations::current_schema_version()),
        default_subscriptions: generated::syncular::default_subscriptions,
        adapter_for: generated::diesel_tables::adapter_for,
    }
}
```

`AppSchema` is the contract between generated app code and the generic Rust
runtime. It is how the runtime knows which tables are app tables, which columns
are scopes, which fields are CRDT/encrypted/blob-backed, and how to apply
snapshot rows without hard-coded table logic.

## Schema versions

`generated/rust/migrations.rs` carries the app migration list and current
schema version. The client sends that schema version during sync so the server
can accept, transform, or reject older clients deliberately.

If your server cannot safely handle a client schema version, return a stable
"please update app" error rather than trying to partially sync an incompatible
shape.

## What not to edit

Do not edit generated files by hand. Change one of these instead:

- SQLite migrations when the local replica shape changes.
- `syncular.app.ts` when Syncular semantics change.
- server table handlers when server projection, auth, or migration-compat
  behavior changes.

Run codegen in `--check` mode in CI so generated code cannot drift from the
contract.
