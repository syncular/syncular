# Mutations

Rust writes go through generated mutation namespaces. This keeps app code typed
without allowing raw synced-table writes that bypass Syncular state.

```rust
use generated::syncular;
use syncular::prelude::SyncularGeneratedMutationsExt;

let inserted = client.mutations().tasks().insert(
    syncular::NewTask::with_generated_id(
        "Write through Syncular",
        "user-rust",
        Some("project-rust"),
    ),
)?;
```

## Update rows

Patch builders are generated per table:

```rust
client
    .mutations()
    .tasks()
    .update(syncular::TaskPatch::new(&inserted.id).completed(1))?;
```

Only changed fields need to be set on the patch. The runtime records changed
fields so events and live queries can refresh precisely.

## Batch a commit

Use `client.commit(...)` when multiple writes should share one local commit and
one outbox entry:

```rust
let result = client.commit(|tx| {
    let id = tx.tasks().insert(syncular::NewTask::with_generated_id(
        "Batched task",
        "user-rust",
        Some("project-rust"),
    ))?;

    tx.tasks()
        .update(syncular::TaskPatch::new(&id).completed(1))?;

    Ok(id)
})?;

println!("queued commit {}", result.commit.client_commit_id);
```

## Leased mutations

Use leased mutations when a user has an offline auth lease for a scope. Leased
mutations attach the lease proof to the queued operation so the server can
validate that the local write was allowed while the device was offline.

```rust
client
    .leased_mutations()
    .tasks()
    .update(syncular::TaskPatch::new(task_id).title("Offline edit"))?;
```

Use normal `mutations()` when the client is online or when the server does not
require an offline lease for the operation.

## Delete behavior

Generated deletes follow the table contract. Tables with soft-delete columns
write a soft-delete mutation. Tables without soft-delete support write a delete
operation.

```rust
client.mutations().tasks().delete(task_id)?;
```

## Sync after mutation

Local mutation commits write SQLite and the outbox immediately. Remote sync is
separate:

```rust
client.sync_http()?;
```

UI apps should usually enqueue mutations through the
[Runtime](/docs/clients/rust/runtime) so SQLite/outbox work runs
on the worker and emits ordered events back to the host.
