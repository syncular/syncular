# Clients

Syncular client docs are grouped by application language. Pick the language
your app code is written in, then use the feature docs for cross-cutting
capabilities such as conflicts, blobs, CRDT fields, encryption, presence, and
offline auth.

## How to choose

- Use [JavaScript](/clients/javascript) when your app layer is
  TypeScript/JavaScript and you want typed Kysely reads plus generated
  Syncular mutation helpers.
- Use [Rust](/clients/rust) when your app layer is Rust or when a native
  shell embeds the Rust runtime directly.
- Use JavaScript host pages for React Native/Expo, Tauri, and Electron when
  the UI is TypeScript but SQLite/sync should stay inside the Rust runtime.

The server remains authoritative in every path. Client docs cover local
replicas, generated code, mutations, runtime lifecycle, and testing; server
handler behavior is documented under [Server](/server).
