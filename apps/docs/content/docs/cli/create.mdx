# Create Commands

`syncular create` covers two distinct flows:

- **libraries mode:** generate `src/syncular/*` modules, config, migration script, and package scripts
- **demo mode:** generate a runnable full demo project

Run commands with one of:

- `syncular ...` (global install)
- `npx @syncular/cli ...`
- `bunx @syncular/cli ...`

## Create libraries

Run the interactive wizard:

```bash
syncular create
```

Run non-interactive:

```bash
syncular create --no-interactive \
  --dir ./my-app \
  --targets server,react,expo \
  --server-dialect postgres \
  --react-dialect wa-sqlite
```

### Targets

Supported `--targets` values:

- `server`
- `react`
- `vanilla`
- `expo`
- `react-native`
- `electron`
- `proxy-api`

If `--targets` is omitted, defaults are `server,react`.

### Dialects

| Target | Dialect flags |
|---|---|
| `server` | `--server-dialect postgres|sqlite` |
| `react` | `--react-dialect wa-sqlite|pglite|bun-sqlite|better-sqlite3|sqlite3` |
| `vanilla` | `--vanilla-dialect wa-sqlite|pglite|bun-sqlite|better-sqlite3|sqlite3` |
| `electron` | `--electron-dialect electron-sqlite|better-sqlite3` |
| `expo` | fixed: `expo-sqlite` |
| `react-native` | fixed: `react-native-nitro-sqlite` |
| `proxy-api` | no dialect flag |

### Generated output (libraries mode)

Always generated:

- `syncular.config.json`
- `scripts/syncular-migrate.ts`
- `scripts/syncular-typegen.ts`
- `package.json` scripts:
  - `db:migrate:status`
  - `db:migrate`
  - `db:migrate:reset`
  - `db:typegen`
  - `db:prepare`

Generated per selected target:

- `src/syncular/server/index.ts`
- `src/syncular/react/index.ts`
- `src/syncular/vanilla/index.ts`
- `src/syncular/expo/index.ts`
- `src/syncular/react-native/index.ts`
- `src/syncular/electron/index.ts`
- `src/syncular/proxy-api/index.ts`

## Create demo

Generate a runnable demo project:

```bash
syncular create demo --dir ./syncular-demo
cd ./syncular-demo
bun install
bun dev
```

What demo mode generates:

- Hono server (`src/server/index.ts`)
- Vite React client (`src/client/*`)
- shared schema + migrations (`src/shared/db.ts`, `scripts/syncular-migrate.ts`)
- `syncular.config.json`

## Flags

Shared create flags:

- `--dir <path>` (default `.`)
- `--force` (overwrite generated files)
