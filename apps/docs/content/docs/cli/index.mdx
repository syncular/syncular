# CLI

The Syncular CLI provides two core workflows:

- scaffold projects (`create --template ...`)
- run migration adapter operations (`migrate-status`, `migrate-up`)

## Run the CLI

Global install:

```bash
syncular --help
```

Run without global install:

```bash
npx @syncular/cli --help
bunx @syncular/cli --help
```

## Command reference

| Command | What it does |
|---|---|
| `syncular create --template syncular-libraries` | Scaffold integration libraries (default template) |
| `syncular create --template syncular-demo` | Scaffold a runnable full demo (Hono + Vite React + WA-SQLite) |
| `syncular create --template spaces-app` | Scaffold a managed Spaces app |
| `syncular create --template spaces-demo` | Scaffold the managed Spaces demo project |
| `syncular migrate-status` | Show current/target/pending migration versions via adapter |
| `syncular migrate-up` | Apply migrations via adapter |
| `syncular doctor` | Run local checks (runtime + workspace + git) |
| `syncular console` | Serve a local Syncular Console instance (use `--server-url`/`--token-env` to preconfigure) |
| `syncular interactive` | Open the full TUI menu |
