# Collaborative Apps

## Use this recipe when

- multiple users work in the same project, room, or workspace
- most product state is durable records, but some fields need richer
  collaboration
- you want presence and fast realtime updates without giving up a
  server-authoritative sync model

## Shape

- shared scope boundary such as `project_id`, `room_id`, or `workspace_id`
- durable records sync through normal table handlers and commit-log pull/push
- presence runs over realtime for ephemeral online state
- Yjs is optional, per field — for rich collaborative subdocuments, not as a
  replacement for the whole data model

## App contract

Documents are normal synced rows; only the collaboratively edited field is a
CRDT field:

```ts
import { defineSyncularClient, scope, syncedTable, yjsText }
  from '@syncular/typegen';

export const app = defineSyncularClient({
  typescriptOutputPath: 'src/generated/syncular.generated.ts',
  typescriptServerOutputPath: 'src/generated/syncular.server.generated.ts',
  tables: {
    documents: syncedTable({
      table: 'documents',
      subscriptionId: 'sub-documents',
      serverVersion: 'server_version',
      scopes: [scope('project_id', { source: 'projectId', required: true })],
      crdt: {
        body: yjsText({
          stateColumn: 'body_yjs_state',
          containerKey: 'body',
        }),
      },
    }),
  },
});
```

## Server: membership scopes, websocket, Yjs plugin

```ts
import { createServerHandler } from '@syncular/server';
import { createSyncServer } from '@syncular/server/hono';
import { createYjsServerPushPlugin } from '@syncular/server/crdt-yjs';
import { upgradeWebSocket } from 'hono/bun';

const { syncRoutes } = createSyncServer<ServerDb, Auth>({
  db,
  dialect,
  sync: {
    handlers: [
      createServerHandler<ServerDb, AppDb, 'documents', Auth>({
        table: 'documents',
        scopes: ['project:{project_id}'],
        resolveScopes: async (ctx) => ({
          project_id: await memberProjectIds(ctx.db, ctx.actorId),
        }),
      }),
    ],
    plugins: [
      createYjsServerPushPlugin({
        rules: [
          {
            table: 'documents',
            field: 'body',
            stateColumn: 'body_yjs_state',
            kind: 'text',
          },
        ],
      }),
    ],
    authenticate: (request) => verifyRequest(request),
  },
  routes: {
    websocket: {
      allowedOrigins: ['https://app.example.com'],
      heartbeatIntervalMs: 15_000,
    },
  },
  upgradeWebSocket,
});
```

The websocket delivers wake-ups and deltas for changed project scopes; the
Yjs plugin merges pushed editor updates into the canonical state column
server-side (`server-merge` mode). For end-to-end encrypted documents, use
`syncMode: 'encrypted-update-log'` instead — see
[Encryption](/features/field-encryption).

## Client: realtime, presence, editor binding

```ts
// Subscribe to the open project and start realtime.
await syncular.setSubscriptions([
  documentSubscription({ actorId: userId, projectId }),
]);
await syncular.sync();
await syncular.start();

// Presence: who is in this project, and what are they editing?
syncular.presence.join(`project:${projectId}`, { documentId });
const stopPresence = syncular.presence.onChange((event) => {
  renderAvatars(event.scopeKey, event.presence);
});

// Durable record edits are ordinary generated mutations.
await syncular.mutations.documents.update(documentId, { title: nextTitle });
```

Bind the rich-text field to the editor through
`@syncular/client/crdt-yjs`, which connects a Yjs `Y.Doc` to the row
field with queued update flushing and backpressure handling — the editor
(TipTap/ProseMirror or any Yjs-compatible editor) never blocks on sync. See
[CRDT Fields](/features/crdt-fields#client-apis).

## What stays durable vs. ephemeral

| State | Where it lives |
|---|---|
| Documents, titles, membership, comments | Synced rows (commit log) |
| Rich text body | CRDT field (Yjs document) |
| Who is online / editing | Presence (never persisted) |
| Cursors and selections | Editor layer; presence metadata only if others need it |

## Watch-outs

- Separate durable product records from ephemeral presence or cursor state.
- Test reconnect and concurrent-edit flows early: backoff, catch-up pulls,
  presence rejoin, and Yjs convergence after a long offline gap.
- Compact CRDT update logs (`compactCrdtField`) so long-lived documents do
  not accumulate unbounded history.
- If your whole product is effectively one CRDT document, compare this shape
  against [Jazz](/start/comparison/vs-jazz) before committing.
