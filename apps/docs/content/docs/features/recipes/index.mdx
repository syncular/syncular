# Common Architectures

Use these recipes when you want a starting shape instead of assembling the system one capability at a time.

Each recipe focuses on:

- the main scope boundaries
- the capabilities to prioritize first
- the docs you should read next

Before choosing a recipe, it helps to skim [How Sync Works](/learn/architecture) and [Scopes](/learn/scopes).

  <Card title="Multi-Tenant SaaS" href="/features/recipes/multi-tenant-saas">

  </Card>
  <Card title="Offline Field App" href="/features/recipes/offline-field-app">
    Durable local state, long offline windows, and eventual sync for mobile or desktop work
  </Card>
  <Card title="Blobs and Media" href="/features/recipes/blobs-and-media">

  </Card>
</Cards>

## After you pick a recipe

- Wire the core backend path with [Server Getting Started](/server/getting-started)
- Add a client with [JavaScript](/clients/javascript) or [Rust](/clients/rust)
- Encode the access model in [Table Handlers](/server/table-handlers)
- Validate the shape with [Testing](/testing)
