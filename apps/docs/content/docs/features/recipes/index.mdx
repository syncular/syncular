# Common Architectures

Use these recipes when you want a starting shape instead of assembling the system one capability at a time.

Each recipe focuses on:

- the main scope boundaries
- the capabilities to prioritize first
- the docs you should read next

Before choosing a recipe, it helps to skim [How Sync Works](/docs/learn/architecture) and [Scopes](/docs/learn/scopes).

<Cards>
  <Card title="Multi-Tenant SaaS" href="/docs/features/recipes/multi-tenant-saas">
    Tenant, workspace, and project isolation with operator tooling and audit trails
  </Card>
  <Card title="Offline Field App" href="/docs/features/recipes/offline-field-app">
    Durable local state, long offline windows, and eventual sync for mobile or desktop work
  </Card>
  <Card title="Blobs and Media" href="/docs/features/recipes/blobs-and-media">
    Large catalogs, attachment sync, snapshot chunking, and storage-aware operations
  </Card>
  <Card title="Collaborative Apps" href="/docs/features/recipes/collaborative-apps">
    Shared records, presence, realtime updates, and optional Yjs-backed fields
  </Card>
</Cards>

## After you pick a recipe

- Wire the core backend path with [Server Setup](/docs/server/setup-with-hono)
- Add a client with [JavaScript](/docs/javascript) or [Rust](/docs/rust)
- Encode the access model in [Table Handlers](/docs/server/table-handlers)
- Validate the shape with [Testing](/docs/testing)
