# Installation

> **Status:** Syncular is **alpha**. npm packages use the `@syncular` scope and
> Rust crates use the `syncular-*` crate names. Pin versions across npm and
> Cargo, and regenerate clients when your app schema changes.

Use this page to install building blocks. For the implementation flow, choose
[JavaScript](/clients/javascript) or [Rust](/clients/rust), then wire
the [Server](/server).

## JavaScript app path

Use this path when your app code is TypeScript/JavaScript and local reads use
Kysely. Browser/React apps still use the Rust-owned SQLite runtime through
`@syncular/client`.

```bash
# Server
npm install @syncular/server @syncular/server-hono @syncular/server-dialect-postgres
npm install kysely pg hono

# Client and React hooks
npm install @syncular/client @syncular/react kysely

# App contract and generated TypeScript client
npm install -D @syncular/typegen
```

For React Native/Expo, Tauri, or Electron hosts, add the matching host bridge:

```bash
npm install @syncular/client-react-native
npm install @syncular/client-tauri
```

Electron apps normally use `@syncular/client` plus an app-owned preload/main
bridge.

## Rust app path

Use this path when app logic is Rust or a native shell embeds the Rust runtime
directly. Diesel is the Rust query builder; generated mutations keep writes in
Syncular's outbox/conflict path.

```bash
cargo add syncular syncular-client
cargo add diesel --features sqlite,returning_clauses_for_sqlite_3_35
cargo add serde serde_json uuid

# Test fixtures for generated Rust/native clients
cargo add --dev syncular-testkit

# Codegen CLI for app schemas
cargo install syncular-codegen --version 0.1.0
```

Rust apps still define the app contract with `@syncular/typegen` at development
time so JavaScript, Rust, and native bindings consume the same generated schema
handoff:

```bash
npm install -D @syncular/typegen
```

## Server packages

| Package | Description |
|---------|-------------|
| `@syncular/server` | Core sync engine (push, pull, pruning, compaction) |
| `@syncular/server-hono` | Hono adapter with HTTP routes, WebSocket, and console |
| `@syncular/server-dialect-postgres` | Postgres and Neon-backed Postgres dialect for sync SQL (`createPostgresServerDialect()` / `createNeonServerDialect()`) |
| `@syncular/server-dialect-sqlite` | SQLite dialect for sync SQL |
| `@syncular/server-cloudflare` | Cloudflare Workers + Durable Objects adapter |

### Server database drivers

Server handlers use Kysely and a server database driver. This is separate from
the client SQLite runtime.

```bash
# Postgres (production)
npm install kysely pg

# SQLite server dialect for development/testing servers
npm install @syncular/dialect-better-sqlite3

# Neon-backed server runtimes
npm install @syncular/dialect-neon
```

## Client packages

| Package | Description |
|---------|-------------|
| `@syncular/client` | Rust-owned browser client package with TypeScript bindings, worker runtime, Kysely local DB, blob APIs, encryption configuration, and CRDT integration |
| `@syncular/react` | React hooks and provider for the Rust-owned client |
| `@syncular/client-crdt-adapters` | Optional Yjs/editor adapters above the Rust client CRDT field APIs |

## Rust crates

| Crate | Description |
|-------|-------------|
| `syncular` | Convenience crate and generated Rust extension traits |
| `syncular-client` | Developer-facing Rust SDK for typed reads, generated writes, sync, conflicts, and live queries |
| `syncular-runtime` | Runtime internals for SQLite, protocol, worker events, blobs, CRDT fields, encryption, and realtime |
| `syncular-codegen` | App-schema code generator for generated Rust clients |
| `syncular-testkit` | Rust/native test fixtures, stateful app server, transports, and assertions |
| `syncular-protocol` | Shared protocol structures for runtime/server integration work |

## Optional packages

```bash
# Optional client editor adapters above Syncular CRDT fields
npm install @syncular/client-crdt-adapters yjs

# Edge relay server
npm install @syncular/relay

# Type generation from server schema
npm install @syncular/typegen
```

## TypeScript setup

Syncular is written in TypeScript and ships full type definitions. For best results, enable strict mode:

```json
{
  "compilerOptions": {
    "strict": true,
    "skipLibCheck": true
  }
}
```
