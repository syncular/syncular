# Quick Start

> **Alpha.** APIs, wire formats, and storage layouts will change between releases. Pin versions and expect manual migrations between versions.

The fastest way to a running Syncular app is the scaffolder:

```bash
bunx create-syncular-app my-app
# or: npm create syncular-app@latest my-app

cd my-app
bun install
bun dev
```

`bun dev` starts a Bun + Hono sync server (SQLite, port 4100) and a Vite dev
server side by side. Open the printed URL and you have a local-first todo app:
writes land in local SQLite instantly, queue in the outbox while offline, and
sync to the server when connected.

If you would rather build the same app by hand to see every moving part, follow
[Hello World](/start/hello-world) instead — it walks through the identical
stack file by file.

## What you just scaffolded

```
my-app/
├── syncular.app.ts            # app contract: tables → subscriptions + scopes
├── migrations/0001_initial/   # plain SQL schema (up.sql / down.sql)
├── src/generated/             # committed generated client + server modules
├── src/server/sync-server.ts  # Hono sync server on Bun + SQLite
├── src/client/syncular.ts     # opens the local DB, starts the sync lifecycle
└── src/app.tsx                # React UI on @syncular/client/react live queries
```

The pieces flow in one direction:

1. **Migrations** (`migrations/*.sql`) define the SQLite schema — plain SQL,
   no DSL.
2. **The app contract** (`syncular.app.ts`) maps each synced table to a
   subscription and its [scopes](/learn/scopes) — the things SQL cannot
   express.
3. **Generate** (`bun run codegen`, i.e. `npx syncular generate`) turns both
   into typed client and server modules under `src/generated/`. They are
   committed, so the app runs without a codegen toolchain; regenerate after
   any schema or contract change. See
   [Generated Client](/clients/javascript/generated-client).
4. **The server** (`src/server/sync-server.ts`) registers a generated table
   handler with `@syncular/server/hono` and owns the commit log and
   authorization. See [Server Getting Started](/server/getting-started).
5. **The client** (`src/client/syncular.ts`) calls the generated
   `createSyncularAppDatabase(...)`, subscribes, and starts the sync
   lifecycle; the React UI reads with live queries and writes through
   generated mutations. See the
   [JavaScript client guide](/clients/javascript).

The scaffolded auth is a single hard-coded demo user — replace
`authenticate()` on the server and `getHeaders` on the client before anything
leaves your machine.

## Evolve it

- Add a column or table: write a new migration, update `syncular.app.ts` if
  subscriptions/scopes change, run `bun run codegen`, restart.
- Keep `bun run codegen:check` (the `--check` form) in CI so generated files
  never drift from the contract.
