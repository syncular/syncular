# Hello World

> **Alpha.** APIs, wire formats, and storage layouts will change between releases. Pin versions and expect manual migrations between versions.

<Callout title="Fastest start">
Scaffold everything on this page with one command, then run it:

```bash
bunx create-syncular-app my-app
cd my-app && bun install && bun dev
```

The rest of this page walks through the same app manually so you see every
moving part once.
</Callout>

This walkthrough builds the smallest real Syncular app: a `tasks` table that
syncs between a Bun + Hono server and a browser client. It is adapted directly
from the runnable demo app in the repository (`apps/demo`); run `bun run demo`
in a clone to see the finished version with two side-by-side clients.

## 1. Install

```bash
# Client
npm install @syncular/client kysely

# Server
npm install @syncular/server @syncular/server-hono @syncular/server-dialect-sqlite @syncular/core @syncular/dialects hono

# CLI and app-contract definitions (dev-time only)
npm install -D syncular @syncular/typegen
```

The `syncular` package is the CLI (`npx syncular generate`) — application code
always imports from the scoped `@syncular/*` packages. The example server uses
SQLite so there is nothing else to provision; production servers use Postgres
via `@syncular/server-dialect-postgres`.

## 2. Define the schema and app contract

Create `migrations/0001_initial/up.sql`:

```sql
CREATE TABLE IF NOT EXISTS tasks (
  id TEXT PRIMARY KEY,
  title TEXT NOT NULL,
  completed INTEGER NOT NULL DEFAULT 0,
  user_id TEXT NOT NULL,
  created_at BIGINT NOT NULL DEFAULT 0,
  server_version BIGINT NOT NULL DEFAULT 0
);
```

Create `syncular.app.ts` — this maps the table to a subscription and a scope:

```typescript
import { defineSyncularClient, scope, syncedTable } from '@syncular/typegen';

export const app = defineSyncularClient({
  typescriptOutputPath: 'src/generated/syncular.generated.ts',
  typescriptServerOutputPath: 'src/generated/syncular.server.generated.ts',
  tables: {
    tasks: syncedTable({
      table: 'tasks',
      subscriptionId: 'sub-tasks',
      scopes: [scope('user_id', { source: 'actorId', required: true })],
      serverVersion: 'server_version',
    }),
  },
});
```

Generate the typed client and server modules:

```bash
npx syncular generate --manifest-dir .
```

## 3. Start the server

The server owns the commit log and decides who may sync what. Create
`server.ts` (run it with `bun run server.ts`):

```typescript
import { createDatabase } from '@syncular/core';
import { createBunSqliteDialect } from '@syncular/dialects/bun-sqlite';
import {
  createServerHandler,
  ensureSyncSchema,
  type SyncCoreDb,
} from '@syncular/server';
import { createSqliteServerDialect } from '@syncular/server-dialect-sqlite';
import { createSyncServer } from '@syncular/server-hono';
import { Hono } from 'hono';
import {
  type SyncularAppDb,
  syncularGeneratedCodecs,
} from './src/generated/syncular.generated';

interface ServerDb extends SyncCoreDb, SyncularAppDb {}
type AppAuth = { actorId: string };

const dialect = createSqliteServerDialect();
const db = createDatabase<ServerDb>({
  dialect: createBunSqliteDialect({ path: 'hello-world.sqlite' }),
  family: 'sqlite',
});

await ensureSyncSchema(db, dialect);
// Apply migrations/0001_initial/up.sql to `db` with your migration runner.

const { syncRoutes } = createSyncServer<ServerDb, AppAuth>({
  db,
  dialect,
  sync: {
    handlers: [
      createServerHandler<ServerDb, SyncularAppDb, 'tasks', AppAuth>({
        table: 'tasks',
        scopes: ['user:{user_id}'],
        codecs: syncularGeneratedCodecs,
        resolveScopes: async (ctx) => ({ user_id: [ctx.actorId] }),
      }),
    ],
    authenticate: (request) => {
      const token = request.headers
        .get('authorization')
        ?.replace('Bearer ', '');
      // Hello-world auth: one hard-coded user. Replace with real session
      // verification before anything leaves your machine.
      return token === 'demo-user' ? { actorId: 'demo-user' } : null;
    },
  },
});

const app = new Hono().route('/sync', syncRoutes);

export default { port: 4101, fetch: app.fetch };
```

## 4. Open the client, write, and read

The generated module exports a typed database factory and subscription
helpers. In your browser app:

```typescript
import {
  getSyncularRuntimeArtifact,
  SyncularClientLifecycle,
} from '@syncular/client';
import {
  createSyncularAppDatabase,
  taskSubscription,
} from './generated/syncular.generated';

const actorId = 'demo-user';

const database = await createSyncularAppDatabase({
  config: {
    baseUrl: 'http://127.0.0.1:4101/sync',
    actorId,
    clientId: 'hello-world-client',
    fileName: 'hello-world.sqlite',
    storage: 'indexedDb',
  },
  getHeaders: async () => ({ authorization: 'Bearer demo-user' }),
  runtimeArtifacts: [getSyncularRuntimeArtifact('full')],
  subscriptions: [taskSubscription({ actorId })],
});

// Push/pull loop. Add `realtime` options for websocket deltas instead of polling.
const lifecycle = new SyncularClientLifecycle(database.client, {
  pollIntervalMs: 5_000,
});
await lifecycle.start();

// Write through the generated mutation — applied to local SQLite instantly,
// queued in the outbox, pushed to the server when online.
await database.mutations.tasks.insert({
  id: crypto.randomUUID(),
  title: 'Hello, Syncular',
  completed: 0,
  user_id: actorId,
  created_at: Date.now(),
});

// Read from local SQLite with Kysely — no network round-trip.
const tasks = await database.db
  .selectFrom('tasks')
  .selectAll()
  .orderBy('created_at', 'desc')
  .execute();

console.log(tasks);
```

That is the whole loop: the write lands in local SQLite immediately, the
outbox pushes it to the server's commit log, and every other client of the
same user pulls it. Kill the network and the app keeps working; reconnect and
it catches up.

React apps wrap the same client in `SyncProvider` and use `useSyncQuery` /
`useMutations` from [`@syncular/react`](/clients/javascript/react) for live,
re-rendering queries.
