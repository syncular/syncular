# API Reference

The Syncular server exposes two API families:

- `sync` endpoints for client synchronization and product-facing audit history
- `console` endpoints for operator dashboards, forensics, and maintenance workflows

All paths below are shown as they appear in the OpenAPI docs (i.e. mounted at `/sync` and `/console/*`).

## Choose the right API family

| Need | Surface |
|---|---|
| Push and pull application data | `POST /sync` |
| Show recent sync history inside your product | `/sync/audit/*` |
| Upload and download blobs | `/sync/blobs/*` |
| Investigate incidents or inspect request payloads | `/console/events*`, `/console/timeline`, `/console/commits` |
| Inspect retained Rust client runtime evidence | `/console/client-diagnostics*` |
| Run maintenance actions | `/console/prune*`, `/console/compact`, `/console/notify-data-change` |
| Manage operator credentials | `/console/api-keys*` |

For human workflows, use [Operate](/docs/operate). For product-facing history UX, use [Audit & History](/docs/features/audit-and-history).

## Base Endpoints

### Sync Endpoints

| Method | Path | Description |
|--------|------|-------------|
| `POST` | `/sync` | Combined sync (`push` and/or `pull`) |
| `GET` | `/sync/audit/commits` | List commit audit history (auth/partition scoped) |
| `GET` | `/sync/audit/commits/{commitSeq}` | Get one commit with operation-level changes |
| `GET` | `/sync/audit/rows/{table}/{rowId}` | Read scoped row audit history |
| `GET` | `/sync/audit/debug-export` | Export a redacted sync debug bundle |
| `GET` | `/sync/realtime` | WebSocket realtime connection for verified deltas and pull-required wake-ups (not part of OpenAPI) |
| `GET` | `/sync/snapshot-artifacts/{artifactId}` | Download a verified scoped bootstrap snapshot artifact |
| `GET` | `/sync/snapshot-chunks/{chunkId}` | Download snapshot chunks for large bootstraps |
| `POST` | `/sync/blobs/upload` | Initiate a blob upload (optional feature) |
| `POST` | `/sync/blobs/{hash}/complete` | Mark a blob upload complete |
| `GET` | `/sync/blobs/{hash}/url` | Get a signed download URL for a blob |

### Console Endpoints

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/console/stats` | Get server stats |
| `GET` | `/console/commits` | Browse commit history |
| `GET` | `/console/clients` | List connected clients |
| `POST` | `/console/client-diagnostics` | Ingest a redacted Rust client diagnostic snapshot |
| `GET` | `/console/client-diagnostics` | List latest redacted Rust client diagnostic snapshots |
| `GET` | `/console/client-diagnostics/{id}` | Get the latest redacted snapshot for one client |
| `GET` | `/console/client-diagnostics/{id}/history` | Page retained redacted snapshots for one client |
| `GET` | `/console/handlers` | List table handlers |
| `GET` | `/console/row-history/{table}/{rowId}` | Read retained row history |
| `GET` | `/console/row-investigation/{table}/{rowId}` | Investigate why a row is or is not visible |
| `GET` | `/console/debug-export` | Export a redacted console debug bundle |
| `GET` | `/console/storage` | List storage/blob objects (optional, requires `console.blobBucket`) |
| `GET` | `/console/storage/{key}/download` | Download a storage/blob object (optional) |
| `DELETE` | `/console/storage/{key}` | Delete a storage/blob object (optional) |

## Endpoint groups in this reference

- **Sync and Audit**: client sync, audit history, row audit, debug export, and
  snapshot artifacts/chunks
- **Blobs**: blob upload and download flow
- **Console Read Model**: stats, commits, clients, diagnostics, handlers,
  timeline, row history, row investigation, and debug export
- **Console Events and Storage**: request events, payload snapshots, and object storage
- **Console Auth**: API keys and token lifecycle
- **Console Maintenance**: notify-data-change, prune, and compact

## Authentication

Syncular does not prescribe an authentication mechanism. The server calls your `authenticate()` function and uses whatever you decide (Bearer tokens, cookies, session headers, etc).

Typical Bearer token auth looks like:

```http
Authorization: Bearer <your-token>
```

## Request/Response Format

All endpoints use JSON for request and response bodies:

```http
Content-Type: application/json
Accept: application/json
```

## OpenAPI Specification

The repo includes a generated OpenAPI spec:

- **Source**: `packages/server-hono/openapi.json`

If you want to serve it at runtime, mount the handler:

```ts
import { createOpenAPIHandler } from '@syncular/server-hono';

app.get('/openapi.json', createOpenAPIHandler(app, { title: 'Syncular API' }));
```

If you also want a Scalar reference page, use the paired helper:

```ts
import { createOpenAPIDocsRoutes } from '@syncular/server-hono';

app.route('/', createOpenAPIDocsRoutes(app, { title: 'Syncular API' }));
```

This serves both:
- `/openapi.json`
- `/spec`

### Regenerating OpenAPI

To regenerate the OpenAPI specification:

```bash
bun generate:openapi
```

This will update the `openapi.json` file based on the current route definitions.

`GET /sync/realtime` is intentionally not part of the OpenAPI document because
it is a WebSocket channel rather than a JSON HTTP endpoint. Rust-first clients
use it for verified realtime sync-pack deltas when possible and explicit
pull-required wake-ups when HTTP recovery is needed.
