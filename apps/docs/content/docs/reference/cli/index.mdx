# CLI

The `syncular` npm package is **CLI-only**. It currently ships exactly two
commands:

```text
usage: syncular <command>

commands:
  generate [--check] [--manifest-dir <path>] [--migrations-dir <path>] [--rust-output-dir <path>] [--app <path>]
  codegen install [--version <version>] [--root <path>] [--force]
```

Run it with a global install (`syncular --help`) or without one
(`npx syncular --help`, `bunx syncular --help`). The help output is the
source of truth for available commands and flags.

## Command reference

| Command | What it does |
|---|---|
| `syncular generate` | Generate the Syncular app handoff (`generated/syncular.codegen.json`) and Rust/TypeScript generated app code |
| `syncular generate --check` | Verify generated output is current without rewriting files (CI) |
| `syncular codegen install` | Install the Rust `syncular-codegen` binary with Cargo into Syncular's tool cache |

See [Generate Command](/reference/cli/generate) for the full pipeline, flags,
and environment variables.

## Scaffolding new projects

Project scaffolding is a separate package, not a `syncular` subcommand:

```bash
bunx create-syncular-app my-app
```

It scaffolds a runnable local-first app (migrations, `syncular.app.ts`,
server, client). The [Quick Start](/start/quick-start) tours the generated
layout.

## What the CLI does not do

Earlier prototypes shipped `create`, `migrate`, `console`, `dev`, and managed
Spaces commands under `syncular ...`. Those do not exist in the current CLI:

- **Scaffolding** → `bunx create-syncular-app` (above)
- **Migrations** → your app owns its migration runner; see
  [Schema Migrations](/features/migrations)
- **Console** → embed or serve `@syncular/console` assets yourself; see
  [Console Integration](/operate/console-integration)
