# Packages And Crates

Use this page as a package index. Guided setup lives under
[Installation](/start/installation), [JavaScript](/clients/javascript),
and [Rust](/clients/rust).

For a normal browser app, install `@syncular/client` plus whatever UI package
your app already uses, then import the generated app module from
`src/generated/syncular.generated.ts`. Add subpath peers only when you import
that subpath: React for `@syncular/client/react`, Yjs for
`@syncular/client/crdt-yjs`, Sentry for `@syncular/client/sentry`, S3 for
`@syncular/server/s3`, and so on.

## Scaffolding and CLI

| Package | Use |
|---|---|
| `create-syncular-app` | Project scaffolder: `bunx create-syncular-app my-app` (see [Quick Start](/start/quick-start)) |
| `syncular` | CLI-only package: `npx syncular generate`, `npx syncular schema check`, `npx syncular ops check`, and `npx syncular codegen install`. It re-exports nothing â€” import from the scoped `@syncular/*` packages |

## Client package surfaces

| Package / subpath | Use |
|---|---|
| `@syncular/client` | Rust-owned browser worker runtime, Kysely local DB, generated TypeScript client integration, blobs, encryption, CRDT fields, and diagnostics |
| `@syncular/client/react` | React provider, query hooks, mutation hooks, status hooks, presence, and conflict helpers |
| `@syncular/client/react-native` | React Native/Expo bridge package for JavaScript UI code talking to the native runtime |
| `@syncular/client/tauri` | Tauri bridge package for TypeScript UI code talking to the native runtime |
| `@syncular/client/crdt-yjs` | Optional Yjs/editor adapters above Syncular CRDT field APIs |
| `@syncular/client/sentry` | Browser Sentry telemetry adapter; install `@sentry/react` in apps that import it |
| `@syncular/core` | Shared protocol types, error definitions, scopes, sync-pack codecs â€” dependency of everything else |
| `@syncular/core/http` | HTTP sync transport used by clients and the relay |
| `@syncular/core/sentry` | SDK-neutral Sentry telemetry adapter shared by the client and server Sentry bridges |

## Server package surfaces

| Package / subpath | Use |
|---|---|
| `@syncular/server` | Core sync engine, `createServerHandler`, push/pull, pruning, compaction, snapshots |
| `@syncular/server/hono` | Hono HTTP/WebSocket adapter (`createSyncServer`), OpenAPI routes, console routes |
| `@syncular/server/postgres` | Postgres/Neon sync SQL dialect |
| `@syncular/server/sqlite` | SQLite sync SQL dialect |
| `@syncular/server/better-sqlite3`, `/bun-sqlite`, `/d1`, `/libsql`, `/neon`, `/pglite`, `/sqlite3` | Kysely runtime dialect helpers. Drivers are optional peer dependencies |
| `@syncular/server/cloudflare` | Cloudflare Workers/Durable Objects/R2 helpers |
| `@syncular/server/cloudflare/sentry` | Cloudflare Sentry telemetry and worker wrapper helpers; install `@sentry/cloudflare` in apps that import it |
| `@syncular/server/service-worker` | Service-worker server/runtime + wake transport helpers |
| `@syncular/server/relay` | Edge relay for distributed sync topologies |
| `@syncular/server/filesystem`, `/s3` | External snapshot/blob storage adapters |
| `@syncular/server/crdt-yjs` | Optional Yjs CRDT merge helpers for server-side CRDT fields |

## Tooling and operations packages

| Package | Use |
|---|---|
| `@syncular/typegen` | App contract (`syncular.app.ts`) and generated client handoff |
| `@syncular/migrations` | Database migration utilities |
| `@syncular/testkit` | TypeScript/server/browser testing fixtures and helpers |
| `@syncular/console` | Embeddable operator console UI (static assets + components) |

## Rust crates

| Crate | Use |
|---|---|
| `syncular` | Convenience crate and generated Rust extension traits |
| `syncular-client` | Developer-facing Rust SDK for typed reads, generated writes, sync, conflicts, and live queries |
| `syncular-runtime` | Runtime internals: SQLite, protocol, worker events, blobs, CRDT fields, encryption, and realtime |
| `syncular-codegen` | Rust app-client code generator (installed by `npx syncular codegen install`) |
| `syncular-testkit` | Rust/native fixtures, stateful app server, transports, assertions, and conformance data |
| `syncular-protocol` | Shared protocol structures for runtime/server integration work |
