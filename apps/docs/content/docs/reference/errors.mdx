# Error Codes

Every Syncular error carries a stable machine-readable code. This page is the
canonical catalog — other docs link here instead of re-describing codes.

The definitions below are generated from `SYNCULAR_ERROR_DEFINITIONS` in
`@syncular/core` (`packages/core/src/error-responses.ts`), which both the
server and the client classification logic share.

## The error envelope

Server HTTP error responses (and rejected operation results) use one JSON
shape, built by `createSyncularErrorResponse()`:

```json
{
  "error": "sync.version_conflict",
  "code": "sync.version_conflict",
  "message": "The operation conflicts with a newer server row version.",
  "category": "conflict",
  "retryable": false,
  "recommendedAction": "resolveConflict",
  "details": { "target": "...", "issues": [] }
}
```

- `code` (and the legacy-compatible `error` alias) is the stable identifier.
- `category` groups codes for coarse handling (`auth-required`, `forbidden`,
  `invalid-request`, `schema-mismatch`, `conflict`, `not-found`,
  `rate-limited`, `scope-revoked`, `integrity-rejected`, `offline`,
  `limit-exceeded`, `transport`, `storage`, `blob`, `server`, `internal`).
- `retryable` says whether retrying the same request can ever succeed
  (possibly after refreshing auth or waiting).
- `recommendedAction` is a hint for automated handling — see the table below.
- `details` is optional structured context (validation issues, limits, the
  upstream server error for client-side wrappers, …).

### How the client surfaces errors

`@syncular/client` wraps failures in `SyncularClientError`
(`packages/client/src/errors.ts`), which exposes the same `code`, `category`,
`retryable`, `recommendedAction`, and `details` fields. Classification order:

1. A Syncular error envelope embedded in the transport error is parsed and
   used verbatim.
2. Bare HTTP 401/403 map to `sync.auth_required` / `sync.forbidden`.
3. Network-unreachable/offline failures map to `sync.offline`.
4. Runtime schema failures map to `sync.schema_mismatch`; integrity failures
   (hash/byte-length/chain-root mismatches) map to `sync.integrity_rejected`.
5. Anything else stays an ordinary `Error`.

Helpers: `isSyncularOfflineError()`, `syncularErrorStatus()`,
`classifySyncularError()`.

### Recommended actions

| `recommendedAction` | What a handler should do |
|---|---|
| `refreshAuth` | Re-authenticate (refresh token / re-login), then retry |
| `checkPermissions` | Do not retry; the actor lacks access — surface to the user or fix grants |
| `fixRequest` | Do not retry as-is; the request/operation is malformed — fix the caller |
| `resolveConflict` | Run your conflict-resolution flow (see [Conflict Resolution](/learn/conflict-resolution)) |
| `regenerateClient` | Re-run `syncular generate` and ship a client matching the server schema |
| `upgradeClient` | The installed app is too old for this server — prompt an app update |
| `resetClientId` | The client id is unusable for this actor; create a fresh client identity |
| `forceResync` | Discard local sync state for the subscription and bootstrap again |
| `retryLater` | Back off and retry; transient |
| `splitBatch` | Reduce operations per commit and retry |
| `reduceInput` | Shrink the payload/input that exceeded a runtime limit |
| `inspectServer` | Server-side misconfiguration or bug — check server logs/config |
| `inspectStorage` | Local storage (OPFS/SQLite) problem — check quota, persistence, corruption |
| `recreateClient` | The browser worker is wedged — dispose and recreate the client |

## Sync protocol errors (`sync.*`)

Returned by the sync server routes (`POST /`, snapshot/audit routes) and by
push operation results; also produced by client-side classification
(`sync.offline`, `sync.transport_failed`).

### Authentication and auth leases

| Code | Category | Retryable | Action | Cause → fix |
|---|---|---|---|---|
| `sync.auth_required` | auth-required | yes | refreshAuth | `authenticate()` returned null or the token expired → refresh credentials and retry |
| `sync.auth_lease_missing` | auth-required | yes | refreshAuth | A queued offline commit has no auth-lease provenance but the server requires it → refresh auth so replay can attach a lease |
| `sync.auth_lease_invalid` | auth-required | yes | refreshAuth | Lease signature/claims failed verification → issue a fresh lease and retry |
| `sync.auth_lease_expired` | auth-required | yes | refreshAuth | The lease TTL elapsed before the queued commit replayed → refresh auth; the commit replays under the new lease |
| `sync.auth_lease_schema_mismatch` | schema-mismatch | no | regenerateClient | Lease was issued for an incompatible schema version → regenerate and redeploy the client |
| `sync.auth_lease_scope_mismatch` | forbidden | no | checkPermissions | The pushed commit touches scopes the lease does not cover → re-issue with correct capabilities or drop the commit |
| `sync.auth_lease_scope_revoked` | scope-revoked | no | checkPermissions | The scope granted in the lease was revoked since issuance → access removed; handle as revocation |
| `sync.auth_lease_business_rejected` | conflict | no | resolveConflict | Your application lease policy rejected the replayed commit → resolve like a conflict (drop or rework the local change) |

### Request validation and authorization

| Code | Category | Retryable | Action | Cause → fix |
|---|---|---|---|---|
| `sync.forbidden` | forbidden | no | checkPermissions | Row/scope authorization failed (write outside resolved scopes, foreign partition, denied CORS origin) → fix grants or the handler's `resolveScopes` |
| `sync.invalid_request` | invalid-request | no | fixRequest | Body failed schema validation, too many subscriptions, missing snapshot-chunk `scopes`, duplicate subscription ids → see `details.issues` |
| `sync.invalid_client_id` | invalid-request | no | resetClientId | The `clientId` is owned by a different actor → generate a new client identity for this actor |
| `sync.invalid_subscription` | invalid-request | no | fixRequest | Subscription references an unknown table or malformed scope values → fix the subscription request |
| `sync.empty_commit` | invalid-request | no | fixRequest | A pushed commit contains zero operations → don't send empty commits |
| `sync.unknown_table` | schema-mismatch | no | regenerateClient | An operation targets a table with no server handler → register the handler or regenerate the client against the current schema |
| `sync.unsupported_operation` | invalid-request | no | fixRequest | Operation type is not supported for this table → fix the client write path |
| `sync.too_many_operations` | invalid-request | no | splitBatch | A commit exceeded `maxOperationsPerPush` (default 200) → split into smaller commits |
| `sync.constraint_violation` | invalid-request | no | fixRequest | The write violated a NOT NULL / unique / foreign-key constraint → fix the data or schema |

### Sync state and conflicts

| Code | Category | Retryable | Action | Cause → fix |
|---|---|---|---|---|
| `sync.version_conflict` | conflict | no | resolveConflict | `base_version` is older than the server row version → run conflict resolution; the result carries `server_version` and `server_row` |
| `sync.row_missing` | not-found | no | forceResync | Update with `base_version` but the row does not exist on the server → resync the subscription |
| `sync.not_found` | not-found | no | forceResync | Requested sync resource (snapshot chunk/artifact, audit commit) is gone or expired → bootstrap again |
| `sync.scope_revoked` | scope-revoked | no | checkPermissions | A synced subscription's scope was revoked (`status: 'revoked'` on pull) → drop local data for the scope or re-request access |
| `sync.schema_mismatch` | schema-mismatch | no | regenerateClient | Generated client schema is incompatible with the server → regenerate the client |
| `sync.client_schema_unsupported` | schema-mismatch | no | upgradeClient | Client schema version is below `requiredSchemaVersion` → ship an app update |
| `sync.integrity_rejected` | integrity-rejected | no | forceResync | Sync data failed hash/chain-root verification → force a full resync; investigate if recurring |

### Transport, limits, and server state

| Code | Category | Retryable | Action | Cause → fix |
|---|---|---|---|---|
| `sync.offline` | offline | yes | retryLater | The client has no network → sync resumes automatically when connectivity returns |
| `sync.transport_failed` | transport | yes | retryLater | The HTTP/WebSocket transport failed mid-request → retried automatically with backoff |
| `sync.rate_limited` | rate-limited | yes | retryLater | Per-client rate limit hit (defaults: 120 pulls/min, 60 pushes/min) → back off; tune `routes.rateLimit` if legitimate |
| `sync.websocket_not_configured` | server | no | inspectServer | A realtime connection was attempted but no `upgradeWebSocket` is wired → configure websockets or disable realtime |
| `sync.websocket_connection_limit` | rate-limited | yes | retryLater | `maxConnectionsTotal` (5000) or `maxConnectionsPerClient` (3) reached → close stale connections or raise the limits |
| `sync.idempotency_cache_miss` | internal | yes | retryLater | A replayed `clientCommitId` was recognized but its cached result could not be read → retry; check storage health if persistent |
| `sync.missing_scopes` | internal | no | inspectServer | A handler emitted a change without scope metadata → fix the handler's `extractScopes`/emitted changes |

## Client runtime errors (`runtime.*`, `storage.*`, `worker.*`)

Raised locally by the Rust runtime and the browser worker bridge — they never
come from the server.

| Code | Category | Retryable | Action | Cause → fix |
|---|---|---|---|---|
| `runtime.busy` | rate-limited | yes | retryLater | The runtime is saturated (e.g. concurrent exclusive work) → retry shortly |
| `runtime.limit_exceeded` | limit-exceeded | no | reduceInput | A configured runtime limit (payload size, row count, …) was exceeded → shrink the input or raise the limit in [client configuration](/clients/javascript/client-configuration) |
| `runtime.config_invalid` | invalid-request | no | fixRequest | Invalid client/runtime configuration values → fix the options passed to `createSyncularDatabase` |
| `runtime.codegen_mismatch` | schema-mismatch | no | regenerateClient | Generated client code does not match the runtime → re-run `syncular generate` |
| `runtime.internal` | internal | no | inspectServer | Unexpected internal runtime failure → report with diagnostics |
| `storage.failed` | storage | no | inspectStorage | Local SQLite/OPFS operation failed (quota, eviction, corruption) → check storage persistence and quota |
| `worker.closed` | invalid-request | no | fixRequest | API call after `close()` on the browser worker client → fix call ordering |
| `worker.not_open` | invalid-request | no | fixRequest | API call before the worker finished opening → await readiness |
| `worker.protocol_mismatch` | schema-mismatch | no | regenerateClient | Page and worker bundle were built from different versions → rebuild/redeploy both together |
| `worker.request_timeout` | rate-limited | yes | retryLater | A worker request timed out → retry; raise the timeout if the workload is legitimately slow |
| `worker.failed` | internal | no | recreateClient | The worker crashed → dispose and recreate the client |
| `worker.message_unreadable` | internal | no | recreateClient | The worker sent a message the page could not decode → recreate the client; usually a bundling/version issue |

## Console errors (`console.*`)

Returned by the operator console routes (`/console/*`).

| Code | Category | Retryable | Action | Cause → fix |
|---|---|---|---|---|
| `console.auth_required` | auth-required | yes | refreshAuth | Missing/invalid console bearer token → set `console.token` or `SYNC_CONSOLE_TOKEN` and send it |
| `console.forbidden_origin` | forbidden | no | checkPermissions | The console websocket origin is not in `corsOrigins` → add the origin |
| `console.invalid_request` | invalid-request | no | fixRequest | Malformed console query/body → see `details` |
| `console.schema_unavailable` | server | yes | retryLater | Console tables not migrated yet (e.g. dialect `ensureConsoleSchema` pending) → retry; check migrations |
| `console.not_found` | not-found | no | inspectServer | Requested console resource does not exist → verify the id/key |
| `console.downstream_unavailable` | server | yes | retryLater | A console downstream instance is unreachable → retry; check the instance |
| `console.downstream_invalid_response` | server | no | inspectServer | Downstream returned an invalid payload → version mismatch between instances |
| `console.internal` | internal | no | inspectServer | Unexpected console route failure → check server logs |

## Proxy errors (`proxy.*`)

Returned by websocket proxy fronting (relay/edge deployments).

| Code | Category | Retryable | Action | Cause → fix |
|---|---|---|---|---|
| `proxy.auth_required` | auth-required | yes | refreshAuth | Proxy authentication missing/expired → refresh credentials |
| `proxy.forbidden_origin` | forbidden | no | checkPermissions | Websocket origin not allowed by the proxy → add the origin |
| `proxy.connection_limit` | rate-limited | yes | retryLater | Proxy websocket connection limit reached → back off |

## Blob errors (`blob.*`)

Returned by the blob upload/download routes (`/blobs/*`). See
[Blobs & Encryption](/clients/javascript/blobs-encryption) for the flow.

| Code | Category | Retryable | Action | Cause → fix |
|---|---|---|---|---|
| `blob.invalid_request` | blob | no | fixRequest | Malformed blob request (bad hash, missing fields) → fix the request |
| `blob.storage_not_configured` | blob | no | inspectServer | No blob storage backend configured on the server → wire blob storage |
| `blob.too_large` | blob | no | fixRequest | Blob exceeds the configured size limit → shrink or raise the limit |
| `blob.not_found` | blob | no | fixRequest | No blob with that hash → upload it first |
| `blob.forbidden` | forbidden | no | checkPermissions | The actor cannot access this blob → check the owning scope |
| `blob.invalid_token` | auth-required | yes | refreshAuth | The signed blob token is invalid or expired → request a fresh URL |
| `blob.upload_failed` | blob | yes | retryLater | Upload to the storage backend failed → retry |
| `blob.hash_mismatch` | integrity-rejected | no | fixRequest | Uploaded bytes don't match the declared content hash → re-hash and re-upload |
| `blob.size_mismatch` | blob | no | fixRequest | Uploaded byte length doesn't match the declared size → fix the upload |
