How the tunnel works

Data enters one mouth, traverses the throat, exits the other. No data is lost in transit.

MOUTH A — CLIENT

Write locally

Mutations enter the wormhole as commits. SQLite stores them instantly.

THROAT — SYNC

Traverse

Commits travel through the sync protocol. Push up, pull down. Ordered, conflict-aware.

MOUTH B — SERVER

Merge & radiate

Server merges all timelines. Truth exits to every connected client.

Wormhole properties

Offline-First

Commit-Log

Every packet ordered. Full traversal history.

Scoped Auth

Not all data passes through. Scopes filter.
