Capabilities

Stars emerging
from the cloud

Each feature is a bright point of light born from the foundational infrastructure — ready to power your application.

Offline-first

Work without a connection. Every mutation is captured locally and synced when connectivity returns — seamlessly.

Real-time sync

Changes propagate across all connected clients in milliseconds via WebSocket wake-ups and efficient pull cycles.

Conflict resolution

Deterministic last-write-wins with full history. Every mutation is preserved in the commit log for auditability.

End-to-end encryption

Optional E2EE plugin encrypts data client-side before sync. The server never sees plaintext — your data stays private.

Scoped subscriptions

Define scope keys so each client receives only the data it needs. Fine-grained, efficient, and secure by design.

Blob storage

Upload and sync binary files alongside structured data. Images, documents, and attachments — all in one flow.

Genesis

How new apps
come to life

Like stars condensing from the gas and dust of a nebula, your app emerges from Syncular's infrastructure in a few clear steps.

Phase 01

Define your schema

Declare the tables and shapes your application needs. Syncular handles migrations, versioning, and the underlying commit log automatically.

Phase 02

Connect the engine

Initialize the sync client with a transport — HTTP for pull-based sync, WebSocket for real-time. Both work together seamlessly.

Phase 03

Mutate locally, sync globally

Write to the local database as if you're offline. The outbox captures every change and pushes it to the server when ready — conflict-free.

Phase 04

Your app is alive

Reactive queries keep your UI in sync. Presence channels show who's online. Your app works offline, online, and everywhere in between.
